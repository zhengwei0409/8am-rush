## 1. Text 作为 Game Object

**Text Object** 是 Phaser 里专门用于显示文字的 Game Object。

它和 Sprite 一样，是 Scene 里的第一类公民（first-class citizen）：
- 有 `x`, `y` 坐标
- 有 `alpha`、`scale`、`rotation` 等变换属性
- 可以被 Tween 控制
- 可以设置 `setInteractive()` 让它可以被点击

文字在 Phaser 里是**渲染到 Canvas 上的像素图形**，不是 HTML 的 `<p>` 或 `<div>` 标签。这意味着：
- 文字完全融入游戏画面，和 Sprite 处于同一层
- 样式通过 Phaser 的 API 控制，不是 CSS
- 可以和其他 Game Object 一起被相机（Camera）控制

---

## 2. 创建 Text Object

```javascript
const scoreText = this.add.text(x, y, content, style);
```

四个参数：
1. `x` — 水平位置（像素）
2. `y` — 垂直位置（像素）
3. `content` — 要显示的字符串
4. `style` — 样式对象（字体、大小、颜色等）

例子：
```javascript
const scoreText = this.add.text(16, 16, 'Score: 0', {
    fontSize: '24px',
    color: '#ffffff'
});
```

这会在屏幕左上角（16, 16）显示白色的 "Score: 0" 文字，字号 24px。

和 Sprite 一样，Text Object 的默认 Origin 也是 `(0, 0)`（左上角），不是中心。如果要居中对齐，可以改变 Origin：
```javascript
scoreText.setOrigin(0.5, 0.5);  // 改为以中心点定位
```

## 3. Font Family（字体）

**Font Family** 是字体的名称，决定文字"长什么样"。

```javascript
{ fontFamily: 'Arial' }
{ fontFamily: 'Georgia' }
{ fontFamily: '"Press Start 2P"' }  // 带空格的字体名要加引号
```

**重要限制：** Phaser 的 Text Object 使用的是**系统字体**或**Web 字体**。

- **系统字体**（如 Arial、Courier New）：所有系统都有，但样式普通
- **Web 字体**（如 Google Fonts）：需要在 HTML 里提前加载，然后在 Phaser 里使用

游戏里常用的做法是用**像素字体（pixel font）**，例如 "Press Start 2P"，来配合游戏的像素风格。

如果你指定了一个系统上没有的字体，浏览器会降级（fallback）到默认字体，通常是 Times New Roman 或 Serif。

---

## 4. Font Size（字号）

**Font Size** 控制文字的大小，常用单位是 `px`（像素）。

```javascript
{ fontSize: '16px' }    // 小字，适合说明文字
{ fontSize: '32px' }    // 中等，适合 UI 标签
{ fontSize: '64px' }    // 大字，适合标题
```

字号写法是字符串（不是纯数字），需要带单位 `px`。

**字号 vs scaleX/scaleY 的区别：**
- `fontSize` 改变的是**字体渲染的实际大小**，文字清晰
- `scaleX/scaleY` 是在渲染后缩放整个 Text Object（像素拉伸），大字号缩放会模糊

如果要改字体大小，用 `fontSize` 而不是 `scale`。

## 5. Color and Style（颜色与样式）

样式对象（Style Object）支持多种属性：

```javascript
this.add.text(100, 100, 'Game Over', {
    fontFamily: 'Arial Black',
    fontSize: '48px',
    color: '#ff0000',         // 文字颜色（红色）
    stroke: '#000000',        // 描边颜色（黑色）
    strokeThickness: 4,       // 描边粗细（像素）
    shadow: {
        offsetX: 2,
        offsetY: 2,
        color: '#000000',
        blur: 4,
        fill: true
    }
});
```

| 属性 | 说明 | 示例值 |
|------|------|--------|
| `color` | 文字填充颜色 | `'#ffffff'`（白色）|
| `stroke` | 描边颜色 | `'#000000'`（黑色）|
| `strokeThickness` | 描边粗细（px） | `4` |
| `backgroundColor` | 文字背景色 | `'#333333'` |
| `padding` | 背景色的内边距 | `{ x: 10, y: 5 }` |

颜色格式是 CSS 十六进制颜色字符串（`#RRGGBB`）。

**描边（Stroke）** 在游戏里非常实用：白色文字 + 黑色描边，放在任何背景上都清晰可读。

## 6. Dynamic Text Update（动态更新文字）

游戏里的文字往往需要实时更新，比如分数、计时器、血量。

用 `setText()` 方法更新文字内容：

```javascript
// create() 里
this.scoreText = this.add.text(16, 16, 'Score: 0', {
    fontSize: '24px',
    color: '#ffffff'
});
this.score = 0;

// update() 里，或者事件回调里
this.score += 10;
this.scoreText.setText('Score: ' + this.score);
// 或者用 template literal
this.scoreText.setText(`Score: ${this.score}`);
```

`setText()` 是**替换整个文字内容**。每次调用都会重新渲染文字图形，所以不要在 `update()` 里每帧都调用（除非值真的变了）。

其他常用更新方法：
```javascript
scoreText.setColor('#ff0000');     // 改颜色
scoreText.setFontSize('32px');     // 改字号
scoreText.setVisible(false);       // 隐藏
scoreText.setAlpha(0.5);           // 半透明
scoreText.setPosition(200, 100);   // 改位置
```

## Self-Check

**Q1.** Phaser 的 Text Object 和直接在 HTML 里放一个 `<p>` 标签有什么根本区别？

> **答：** Phaser 的 Text Object 是**渲染在 Canvas 上的像素图形**，完全融入游戏画面，可以和 Sprite 处于同一层、被相机控制、被 Tween 动画操作。HTML 的 `<p>` 标签是 DOM 元素，存在于 HTML 结构里，不在 Canvas 内，无法和游戏对象互动，也不能被 Phaser 的系统（物理、相机、Tween 等）控制。

---

**Q2.** 你创建了一个分数 Text Object，位于屏幕 (16, 16)，初始显示 "Score: 0"。玩家得了 50 分，你应该调用什么方法更新显示？写出调用示例。

> **答：** 使用 `setText()` 方法：
> ```javascript
> scoreText.setText('Score: 50');
> // 或者动态拼接
> scoreText.setText('Score: ' + score);
> ```

---

**Q3.** 为什么游戏里常用"白色文字 + 黑色描边"？这个效果在 Phaser 里怎么实现？

> **答：** 白色文字 + 黑色描边（Stroke）可以让文字在**任何背景颜色**上都保持清晰可读 — 在白色背景上有黑色描边做对比，在黑色背景上有白色本身，在彩色背景上也一样醒目。Phaser 里通过 Style 对象的 `stroke` 和 `strokeThickness` 属性实现：
> ```javascript
> this.add.text(100, 100, 'Health: 100', {
>     color: '#ffffff',
>     stroke: '#000000',
>     strokeThickness: 4
> });
> ```
