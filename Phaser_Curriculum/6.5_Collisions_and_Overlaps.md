## 1. Collision vs Overlap — 核心区别

这是本节最重要的概念，先把区别说清楚：

**Collision（碰撞）：**
- 两个物体**不能互相穿过**
- 物理引擎会把它们推开，让它们分开
- 用于：角色站在地板上、子弹打到墙壁停下来、玩家撞到箱子被阻挡

**Overlap（重叠）：**
- 两个物体**可以互相穿过**，物理引擎不会阻止
- 但 Phaser 会**通知你"它们在重叠"**，让你执行自己的逻辑
- 用于：玩家踩到金币（拾取）、玩家进入危险区域（受伤）、角色走进触发区

一句话总结：
> Collision = 物理引擎阻止穿过 + 通知你
> Overlap = 物理引擎不阻止 + 通知你

## 2. 什么是 Collider（碰撞器）？

**Collider** 是你在 Phaser 里注册的一条规则："这两个对象（或组）之间要进行碰撞检测"。

```javascript
// 语法
this.physics.add.collider(objectA, objectB, callback, processCallback, context);
```

最简单的用法：
```javascript
// 玩家和地板之间的碰撞
this.physics.add.collider(player, ground);
```

这一行告诉 Phaser：每帧都检测 `player` 和 `ground` 之间是否碰撞。如果碰了，物理引擎自动把它们分开（玩家站在地板上，不会穿过去）。

带回调函数：
```javascript
this.physics.add.collider(player, enemy, () => {
    // 玩家碰到敌人时执行这里
    player.takeDamage();
});
```

## 3. 什么是 Overlap（重叠检测）？

**Overlap** 用法和 Collider 几乎一样，只是换了方法名：

```javascript
// 语法
this.physics.add.overlap(objectA, objectB, callback, processCallback, context);
```

例子：
```javascript
// 玩家和金币重叠时拾取金币
this.physics.add.overlap(player, coins, (player, coin) => {
    coin.destroy();       // 金币消失
    this.score += 10;     // 加分
    this.updateScore();
});
```

玩家走到金币上时：
- 物理引擎**不会**阻止玩家穿过金币（两者可以重叠）
- 但 Phaser 检测到重叠，自动调用你的回调函数
- 你在回调里写"拾取"的逻辑

---

## 4. Callback（回调函数）

**Callback** 是"当某件事发生时，自动执行的函数"。

Collider 和 Overlap 都支持传入 callback：

```javascript
this.physics.add.collider(player, enemy, onPlayerHitEnemy);

function onPlayerHitEnemy(player, enemy) {
    // 参数是两个碰撞的对象
    console.log('玩家碰到了敌人');
    player.health -= 1;
    enemy.destroy();
}
```

Callback 的参数是**两个发生碰撞/重叠的具体对象**。当一个 Group（组）里有多个物体时，Phaser 会传入具体碰撞的那个对象，让你知道是哪一个。

```javascript
// enemies 是一个 Group，包含多个敌人
this.physics.add.overlap(player, enemies, (player, enemy) => {
    // enemy 是具体碰到的那一个敌人
    enemy.destroy();
});
```

## 5. Blocking vs Non-blocking

这是 Collision 和 Overlap 区别的物理引擎层面解释：

**Blocking（阻挡）= Collision 的行为**

当 Collider 检测到两个 Body 重叠时，物理引擎计算出一个**分离向量（separation vector）**，把两个 Body 推到刚好不重叠的位置。

```
碰撞前：          碰撞检测：        碰撞后（物理引擎处理）：
[玩家]→→→[地板]   发现重叠！        [玩家]
                                    ─────────[地板]─────────
                                    （玩家被推到地板表面上）
```

**Non-blocking（不阻挡）= Overlap 的行为**

Overlap 只是"检测并通知"，不改变任何 Body 的位置或速度。两个 Body 可以完全重叠，物理引擎不干涉。

## 6. Collision Groups（碰撞组）

在真实游戏里，你通常不是让单个 Sprite 和另一个单个 Sprite 碰撞，而是让**一类物体**和**另一类物体**碰撞。

Phaser 的 **Group（组）** 让你把同类物体管理在一起：

```javascript
// 创建一组平台
const platforms = this.physics.add.staticGroup();
platforms.create(400, 568, 'ground');
platforms.create(600, 400, 'ground');
platforms.create(50, 250, 'ground');

// 创建一组金币
const coins = this.physics.add.group();
coins.create(200, 100, 'coin');
coins.create(300, 100, 'coin');

// 玩家和所有平台碰撞
this.physics.add.collider(player, platforms);

// 玩家和所有金币重叠
this.physics.add.overlap(player, coins, collectCoin);
```

用 Group 的好处：
- 一行代码处理"玩家 vs 所有平台"，不需要为每个平台单独注册 Collider
- 新加入 Group 的物体自动参与碰撞检测

**Collision Groups 是游戏里物理检测的标准用法。**

## Self-Check

**Q1.** Collision 和 Overlap 最根本的区别是什么？

> **答：** Collision 是"阻挡式"碰撞：物理引擎检测到两个 Body 重叠时，会自动把它们推开，防止互相穿过，同时通知你。Overlap 是"穿透式"检测：物理引擎不阻止两个 Body 穿过对方，但检测到重叠时会通知你，让你执行自己的逻辑。一个阻挡，一个不阻挡。

---

**Q2.** 下面哪个场景应该用 Collider，哪个应该用 Overlap？

- (a) 玩家站在地板上，不能穿过
- (b) 玩家走进火焰区域受伤
- (c) 子弹打到墙壁停下来
- (d) 玩家碰到门触发剧情对话

> **答：**
> - (a) **Collider** — 地板必须阻挡玩家，不能让玩家穿过
> - (b) **Overlap** — 玩家可以在火焰区域里站着，只需要检测"是否在里面"并扣血
> - (c) **Collider** — 子弹碰到墙壁后速度归零，需要阻挡
> - (d) **Overlap** — 玩家走进门的触发范围，门不需要阻挡玩家，只需要检测"是否到达"并触发对话

---

**Q3.** Callback 函数的两个参数是什么？为什么用 Group 做碰撞检测时，Callback 的参数很重要？

> **答：** Callback 函数的两个参数是**发生碰撞/重叠的两个具体对象**（例如 `player` 和 `enemy`）。用 Group 做碰撞检测时，Group 里有多个物体（例如 20 个敌人），碰撞发生时 Phaser 会把**具体碰到的那一个对象**传进 Callback，而不是整个 Group。这样你才能对正确的那个对象执行操作（例如销毁具体的那枚金币，而不是所有金币）。
