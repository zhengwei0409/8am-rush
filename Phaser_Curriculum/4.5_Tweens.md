# 4.5 — Tweens

## 大局观 (High-Level Map)

在 4.4 你学了通过**切换图片帧**来产生动画效果。但很多"运动"并不需要切换图片：按钮淡入淡出、物体从 A 点滑到 B 点、UI 弹出时的缩放效果……

这些运动的共同点是：**某个数值从一个值平滑地变化到另一个值**。

这就是 **Tween** 的工作 — 不切换帧，而是**自动插值（interpolation）**一个属性的数值变化。

```
大图：
                        ┌─ 帧动画（Animation）— 切换图片帧
游戏里的"运动效果" ─┤
                        └─ Tween — 平滑改变属性数值
```

Tween 和 Animation 是**完全不同**的两种机制，适用于不同场景。

## 1. 什么是 Tween？

**Tween** 是一个让某个**属性**从**起始值**平滑过渡到**目标值**的动画系统。

"Tween" 这个词来自 "in-between" — 自动计算起点和终点**之间**的所有中间值。

一个简单例子：让一个 Sprite 在 1 秒内从 x=100 移动到 x=400：

```javascript
this.tweens.add({
    targets: sprite,      // 要改变哪个对象
    x: 400,               // 目标值（从当前 x=100 变到 x=400）
    duration: 1000,       // 持续 1000ms（1 秒）
    ease: 'Linear'        // 缓动函数
});
```

Phaser 会自动在每一帧计算出 x 应该是多少，然后更新 Sprite 的位置。你不需要手动写 `update()` 里的移动逻辑。

---

## 2. Property Interpolation（属性插值）

**插值（Interpolation）** 就是：给定起点 A 和终点 B，自动计算 A 到 B 之间的所有中间值。

例如 x 从 100 变到 400，线性插值在 1 秒（假设 60fps）里会生成约 60 个中间值：
```
100, 105, 110, 115, ... 390, 395, 400
```

Tween 可以同时插值**多个属性**：

```javascript
this.tweens.add({
    targets: sprite,
    x: 400,          // 位置变化
    y: 200,          // 位置变化
    alpha: 0,        // 淡出（透明度从 1 变到 0）
    scaleX: 2,       // 水平放大到 2 倍
    duration: 1000
});
```

几乎所有**数值类**的属性都可以被 Tween：
- 位置（`x`, `y`）
- 透明度（`alpha`）
- 缩放（`scaleX`, `scaleY`）
- 旋转（`rotation`）
- 颜色值

---

## 3. Duration（持续时间）

**Duration** 控制 Tween 完成的总时间，单位是**毫秒（milliseconds, ms）**。

```javascript
duration: 500    // 0.5 秒
duration: 1000   // 1 秒
duration: 3000   // 3 秒
```

Duration 越短，变化越快（"咻"的一下）；Duration 越长，变化越慢（缓慢渐变）。

Tween 也支持循环：
```javascript
this.tweens.add({
    targets: sprite,
    alpha: 0,
    duration: 500,
    yoyo: true,     // 播完后倒回去（1→0→1→0...）
    repeat: -1      // 无限重复
});
```

`yoyo: true` 让 Tween 播放完后反向播放，适合做"呼吸"效果或闪烁。

---

## 4. Easing Functions（缓动函数）

**Easing Function** 控制属性变化的**速度曲线**，而不是让它一直匀速变化。

想象你把一本书从桌子左边推到右边：
- **Linear（线性）** — 匀速，机械感
- **Ease In** — 先慢后快，像从静止加速
- **Ease Out** — 先快后慢，像减速停止
- **Ease In Out** — 先慢、中间快、再慢，最自然的感觉
- **Bounce** — 到终点后弹跳一下
- **Elastic** — 像弹簧拉过头再回弹

```
Linear:       ────────────────→
Ease Out:     ══════════───────→  (开始快，结尾慢)
Ease In:      ───────══════════→  (开始慢，结尾快)
Bounce:       ════════════╗↑↓→   (到终点后弹跳)
```

在 Phaser 里用法：
```javascript
ease: 'Linear'
ease: 'Quad.easeOut'
ease: 'Bounce.easeOut'
ease: 'Elastic.easeInOut'
```

Easing 是**纯视觉感受**的调整，不改变起点、终点或持续时间，只改变"快慢节奏"。

好的 Easing 让 UI 动画感觉更自然、有质感；差的 Easing（比如全用 Linear）让游戏感觉廉价。

---

## 5. Tween vs Animation：什么时候用哪个？

这是一个容易混淆的地方，核心区别：

| 维度 | Animation（帧动画）| Tween |
|------|-------------------|-------|
| 原理 | 切换不同的图片帧 | 平滑改变数值属性 |
| 需要素材 | 需要 Sprite Sheet / Atlas | 不需要额外图片 |
| 适用场景 | 角色走路、攻击、死亡 | 移动、淡入淡出、缩放、UI 动效 |
| 控制的是 | 显示哪一帧图片 | 属性的数值 |

**实际游戏中，两者经常同时使用：**

- 角色走路：**Animation** 控制腿部动作帧切换
- 角色从屏幕左边滑入：**Tween** 控制 x 坐标从 -100 变到 200
- 角色死亡：**Animation** 播放死亡动作帧 + **Tween** 同时让 alpha 从 1 变到 0（淡出）

> **记忆口诀：** Animation = 换图片，Tween = 改数字。

## Self-Check

**Q1.** "Interpolation（插值）"在 Tween 里是什么意思？

> **答：** 插值是指给定一个属性的起始值和目标值，自动计算出两者**之间**的所有中间值，使属性从起始值平滑过渡到目标值。例如 x 从 100 到 400，Phaser 会在每一帧自动计算 x 应该是 105、110、115……直到 400，你不需要手动写这个计算逻辑。

---

**Q2.** 下面这个 Tween 会产生什么视觉效果？

```javascript
this.tweens.add({
    targets: button,
    scaleX: 1.2,
    scaleY: 1.2,
    duration: 200,
    yoyo: true,
    repeat: 0,
    ease: 'Quad.easeOut'
});
```

> **答：** 按钮会在 0.2 秒内**快速放大到原来的 1.2 倍**（Ease Out = 先快后慢），然后因为 `yoyo: true`，再用 0.2 秒**缩回原来大小**。`repeat: 0` 表示整个过程（放大+缩小）只播一次。视觉上是一个"按下弹起"的点击反馈效果。

---

**Q3.** 一个角色在走路时，哪些效果应该用 Animation，哪些应该用 Tween？

> **答：**
> - **Animation**：角色腿部、手臂的动作 — 这是图片帧的切换，需要美术预先做好每一帧的图片。
> - **Tween**：角色在地图上从左走到右（x 坐标变化）、受伤时闪白（alpha 短暂降低再恢复）、进入房间时的淡入效果 — 这些是数值属性的平滑变化，不需要额外图片素材。
