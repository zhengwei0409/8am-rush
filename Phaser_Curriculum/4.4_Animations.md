# 4.4 — Animations

## 大局观 (High-Level Map)

在 4.3 你学了 Sprite Sheet 把多帧图片打包成一张大图。但光有图片还不够 — 你需要**定义怎么播放这些帧**，以及**以什么速度、是否循环**。

这就是 **Animation（帧动画）** 的职责：把一系列静态帧，按照你规定的顺序和速度，一帧一帧地切换，产生"动起来"的视觉效果。

```
大图：
[ Sprite Sheet ]
       ↓
[ 定义 Animation（帧序列 + 帧率 + 循环）]
       ↓
[ 在 Sprite 上播放 Animation ]
       ↓
[ 屏幕上看到角色动起来 ]
```

注意：这里说的 Animation 是**帧动画（Frame Animation）**，通过切换图片帧实现。和 4.5 的 Tween（属性插值）是不同的概念。

## 1. 什么是帧动画（Frame-based Animation）？

**帧动画**的原理和老式翻页书一模一样：

```
帧 0: 角色腿向前
帧 1: 角色腿中间
帧 2: 角色腿向后
帧 3: 角色腿中间
帧 0: 角色腿向前（循环）
...
```

每一张图叫一**帧（Frame）**。以足够快的速度切换这些帧，人眼就会感知到"动作"。

Phaser 的 Animation 系统就是：
1. 你告诉它用哪些帧、以什么顺序
2. 你告诉它播放速度（帧率）
3. 你告诉它是否要循环
4. Phaser 负责在每一帧计时，自动切换 Sprite 显示的图片

---

## 2. Animation Sequence（动画序列）

**Animation Sequence** 是动画的核心：**你要按什么顺序播放哪些帧**。

例如，走路动画可能用帧 0、1、2、3：

```javascript
this.anims.create({
    key: 'walk',
    frames: this.anims.generateFrameNumbers('player', { start: 0, end: 3 }),
    ...
});
```

`generateFrameNumbers('player', { start: 0, end: 3 })` 会生成：
```
[{ key: 'player', frame: 0 },
 { key: 'player', frame: 1 },
 { key: 'player', frame: 2 },
 { key: 'player', frame: 3 }]
```

帧不一定要连续，也可以是任意顺序：
```javascript
frames: this.anims.generateFrameNumbers('player', { frames: [0, 2, 1, 3] })
```

---

## 3. Frame Rate（帧率）

**Frame Rate** 决定动画播放的**速度**，单位是 **fps（frames per second，每秒帧数）**。

```javascript
this.anims.create({
    key: 'walk',
    frames: ...,
    frameRate: 8,  // 每秒切换 8 帧
    ...
});
```

帧率的直觉：
- `frameRate: 1` — 每秒换 1 帧，极慢，像幻灯片
- `frameRate: 8` — 每秒换 8 帧，走路、跑步常用
- `frameRate: 24` — 每秒换 24 帧，接近电影流畅度

**注意：** 动画帧率（Animation Frame Rate）和游戏帧率（Game Frame Rate）是两回事。
- 游戏帧率是整个屏幕刷新的频率（通常 60fps）
- 动画帧率是动画序列切换帧的频率（你自己设置，通常 8~24fps）

同一个游戏里，游戏帧率是 60fps，但走路动画可能只有 8fps — 这完全正常。

---

## 4. Looping（循环）

**repeat** 属性控制动画是否循环：

```javascript
this.anims.create({
    key: 'walk',
    frames: ...,
    frameRate: 8,
    repeat: -1   // -1 = 无限循环
});
```

| `repeat` 值 | 效果 |
|------------|------|
| `0` | 播放一次就停止（默认） |
| `1` | 播放两次（原来 1 次 + 重复 1 次） |
| `-1` | **无限循环** |

走路、跑步这类持续动作用 `repeat: -1`。
攻击、死亡这类一次性动作用 `repeat: 0`。

---

## 5. Animation Key

**Animation Key** 是你给每个动画取的唯一名字，就像 Asset Key 一样。

```javascript
this.anims.create({
    key: 'walk',   // ← Animation Key
    ...
});
```

一个游戏里通常有很多动画：

```javascript
this.anims.create({ key: 'walk', ... });
this.anims.create({ key: 'run', ... });
this.anims.create({ key: 'jump', ... });
this.anims.create({ key: 'idle', ... });
this.anims.create({ key: 'attack', ... });
```

Key 必须唯一。播放动画时，你用 key 来指定播哪个：
```javascript
sprite.anims.play('walk');
```

**重要：** Animation 是**全局注册**的，在 `this.anims` 上创建后，整个游戏所有 Scene 的所有 Sprite 都可以使用。动画定义和 Sprite 本身是分开的。

---

## 6. 在 Sprite 上播放动画

动画定义好之后，要在具体的 Sprite 上播放：

```javascript
// 在 create() 里定义动画
this.anims.create({
    key: 'walk',
    frames: this.anims.generateFrameNumbers('player', { start: 0, end: 3 }),
    frameRate: 8,
    repeat: -1
});

// 创建 Sprite
const player = this.add.sprite(200, 300, 'player');

// 播放动画
player.anims.play('walk');
```

常用的动画控制方法：

| 方法 | 作用 |
|------|------|
| `sprite.anims.play('walk')` | 播放名为 'walk' 的动画 |
| `sprite.anims.stop()` | 停止当前动画 |
| `sprite.anims.pause()` | 暂停当前动画 |
| `sprite.anims.resume()` | 继续暂停的动画 |

你也可以在 `update()` 里根据游戏状态切换动画：
```javascript
// 伪代码
if (player.isMoving) {
    player.anims.play('walk', true);  // true = 如果已在播放就不重启
} else {
    player.anims.play('idle', true);
}
```

## Self-Check

**Q1.** 帧动画的基本原理是什么？

> **答：** 帧动画通过**快速切换一系列静态图片**来产生运动的视觉效果，原理和翻页书一样。每张图叫一帧，以足够快的速度（由 frameRate 决定）按顺序切换这些帧，人眼就会感知到连续的动作。

---

**Q2.** 一个走路动画定义了 4 帧（帧 0~3），`frameRate: 8`，`repeat: -1`。请描述它的播放行为。

> **答：** 动画每秒切换 8 帧，意味着每 0.125 秒（125ms）换一帧。4 帧播完（0.5 秒）后，因为 `repeat: -1`，动画会**无限重复**从帧 0 开始循环，持续播放，直到你手动停止。

---

**Q3.** Animation Key 和 Asset Key 有什么相似之处？Animation 定义写在哪里，它是属于单个 Sprite 还是全局的？

> **答：** 两者都是唯一的字符串标识符，用来在代码里引用对应的资源。Animation 定义写在 `this.anims.create()` 里，通常放在 `create()` 方法中。它是**全局的（global）**，不属于某个特定 Sprite — 定义一次后，所有 Sprite 都可以通过 key 来播放这个动画。
