# 7.3 — Code Style Differences

## 大局观 (High-Level Map)

在 7.2 你学了 Phaser 3 和 Phaser 4 的 Setup 差异：Phaser 4 需要 Bundler、用 ES Module Imports、没有全局对象。

这节看**代码本身**长什么样。当你真正写游戏逻辑时，Phaser 3 和 Phaser 4 的代码风格有什么不同？

好消息是：**核心游戏概念完全没变**。Scene 还是 Scene，Sprite 还是 Sprite，`preload()` / `create()` / `update()` 还是一样。不同的只是**写法的表达方式**。

## 1. Explicit Imports（显式引入）

Phaser 3 风格：依赖全局 `Phaser` 对象，不需要 import 任何东西。

```javascript
// Phaser 3 — game.js
// 不需要任何 import，Phaser 已经是全局变量

class GameScene extends Phaser.Scene {
    create() {
        this.add.sprite(100, 100, 'player');
    }
}
```

Phaser 4 风格：每个用到的类都必须显式 import。

```typescript
// Phaser 4 — GameScene.ts
import { Scene } from 'phaser/scene';
import { Sprite } from 'phaser/gameobjects';

class GameScene extends Scene {
    create() {
        this.add.sprite(100, 100, 'player');
    }
}
```

**Explicit Imports 的作用：**
- 文件顶部的 import 列表 = 这个文件的依赖清单
- 一眼就能看出这个文件用了 Phaser 的哪些部分
- 让 Bundler 能正确分析依赖，执行 Tree-shaking

---

## 2. Stronger Typing（更强的类型）

Phaser 3（JavaScript）：变量没有类型声明，运行时才知道类型错了。

```javascript
// Phaser 3 — JavaScript
create() {
    this.player = this.add.sprite(100, 100, 'player');
    // this.player 是什么类型？编辑器不确定
}

update() {
    this.player.setVelocityX(200);  // 如果拼错了方法名，运行时才崩
}
```

Phaser 4（TypeScript）：变量有类型声明，编辑器实时提示错误。

```typescript
// Phaser 4 — TypeScript
import { Sprite } from 'phaser/gameobjects';

private player: Sprite;  // 明确声明类型

create(): void {
    this.player = this.add.sprite(100, 100, 'player');
    // 编辑器知道 player 是 Sprite，自动补全所有 Sprite 的方法
}

update(): void {
    this.player.setVelocityX(200);  // 拼错方法名，编辑器立刻画红线
}
```

**Stronger Typing 对你的实际好处：**
- VS Code 自动补全 Phaser 4 所有 API，不需要查文档
- 传错参数类型时，编辑器立刻报错，不用等运行时崩溃
- 代码可读性更高（每个变量的类型一目了然）

## 3. Namespaced Modules（命名空间模块）

Phaser 3 把所有东西都挂在 `Phaser` 这一个全局命名空间下：

```javascript
// Phaser 3 — 所有东西都在 Phaser.xxx 下
Phaser.Scene
Phaser.GameObjects.Sprite
Phaser.Input.Keyboard.KeyCodes
Phaser.Physics.Arcade.Body
```

Phaser 4 把这些分散到独立的模块（包）里，每个模块有自己的命名空间：

```typescript
// Phaser 4 — 按模块分组
import { Scene } from 'phaser/scene';
import { Sprite } from 'phaser/gameobjects/sprite';
import { KeyCodes } from 'phaser/input/keyboard';
import { Body } from 'phaser/physics/arcade';
```

**对比：**

| Phaser 3 路径 | Phaser 4 模块 |
|--------------|--------------|
| `Phaser.Scene` | `'phaser/scene'` |
| `Phaser.GameObjects.Sprite` | `'phaser/gameobjects/sprite'` |
| `Phaser.Input.Keyboard.KeyCodes` | `'phaser/input/keyboard'` |
| `Phaser.Physics.Arcade` | `'phaser/physics/arcade'` |

层级结构是一样的，只是从"全局对象的属性路径"变成了"模块 import 路径"。

## 4. Scene Lifecycle — 没有变化

这是让人放心的地方：**Phaser 4 的 Scene 生命周期和 Phaser 3 完全一样。**

```typescript
// Phaser 4 — Scene 生命周期
import { Scene } from 'phaser/scene';

class GameScene extends Scene {

    preload(): void {
        // 加载资源 — 和 Phaser 3 一样
        this.load.image('player', 'assets/player.png');
    }

    create(): void {
        // 初始化游戏对象 — 和 Phaser 3 一样
        this.player = this.add.sprite(400, 300, 'player');
    }

    update(): void {
        // 每帧逻辑 — 和 Phaser 3 一样
    }
}
```

`preload()` → `create()` → `update()` 的执行顺序、职责、用法，在 Phaser 4 里没有任何改变。

你在 3.2 学的 Scene Lifecycle 知识，在 Phaser 4 里 100% 适用。

---

## 5. Game Object API — 基本相同

Phaser 4 对 Game Object 的 API 做了一些调整和改进，但**核心用法高度相似**。

```typescript
// Phaser 3
this.add.sprite(x, y, 'key');
this.add.text(x, y, 'Hello', { fontSize: '24px' });
this.physics.add.sprite(x, y, 'key');

// Phaser 4（几乎一样）
this.add.sprite(x, y, 'key');
this.add.text(x, y, 'Hello', { fontSize: '24px' });
this.physics.add.sprite(x, y, 'key');
```

具体方法名和参数可能有细微变化，但整体思路完全相同。

主要的改变集中在：
- 一些方法被重命名（更语义化）
- 一些配置选项的结构调整
- TypeScript 类型让参数约束更严格

**这不是重新学 — 这是查 changelog 找差异。**

## 6. 并排对比：同一个功能的写法

### 场景：创建一个带物理的玩家，用方向键控制左右移动

**Phaser 3（JavaScript）：**

```javascript
class GameScene extends Phaser.Scene {
    constructor() {
        super({ key: 'GameScene' });
    }

    preload() {
        this.load.image('player', 'assets/player.png');
    }

    create() {
        this.player = this.physics.add.sprite(400, 300, 'player');
        this.cursors = this.input.keyboard.createCursorKeys();
    }

    update() {
        if (this.cursors.left.isDown) {
            this.player.body.setVelocityX(-200);
        } else if (this.cursors.right.isDown) {
            this.player.body.setVelocityX(200);
        } else {
            this.player.body.setVelocityX(0);
        }
    }
}
```

**Phaser 4（TypeScript）：**

```typescript
import { Scene } from 'phaser/scene';
import { Sprite } from 'phaser/gameobjects';
import { CursorKeys } from 'phaser/input/keyboard';

class GameScene extends Scene {
    private player: Sprite;
    private cursors: CursorKeys;

    constructor() {
        super({ key: 'GameScene' });
    }

    preload(): void {
        this.load.image('player', 'assets/player.png');
    }

    create(): void {
        this.player = this.physics.add.sprite(400, 300, 'player');
        this.cursors = this.input.keyboard.createCursorKeys();
    }

    update(): void {
        if (this.cursors.left.isDown) {
            this.player.body.setVelocityX(-200);
        } else if (this.cursors.right.isDown) {
            this.player.body.setVelocityX(200);
        } else {
            this.player.body.setVelocityX(0);
        }
    }
}
```

**观察两者的差异：**
- 新增了文件顶部的 `import` 语句
- 类属性有了类型声明（`: Sprite`、`: CursorKeys`）
- 方法有了返回类型声明（`: void`）
- `update()` 里的游戏逻辑**一字不差**

> **核心结论：** Phaser 4 的代码改变的是"工程层面的写法"（import、类型），不是"游戏逻辑"。你在 Phaser 3 学到的所有游戏开发思维，在 Phaser 4 里完全保留。

## Self-Check

**Q1.** Phaser 3 和 Phaser 4 在 Scene Lifecycle（`preload` / `create` / `update`）上有什么区别？

> **答：** 没有区别。`preload()` / `create()` / `update()` 的执行顺序、职责、用法在 Phaser 4 里和 Phaser 3 完全一样。这是两个版本之间刻意保持兼容的核心概念，目的是让 Phaser 3 开发者能平滑迁移。

---

**Q2.** 下面这段 Phaser 3 代码，迁移到 Phaser 4 需要做哪几处改动？（不需要写完整代码，列出需要改的地方即可）

```javascript
class MenuScene extends Phaser.Scene {
    create() {
        this.startButton = this.add.sprite(400, 300, 'button');
        this.startButton.setInteractive();
        this.startButton.on('pointerdown', () => {
            this.scene.start('GameScene');
        });
    }
}
```

> **答：** 需要改动的地方：
> 1. 在文件顶部添加 `import { Scene } from 'phaser/scene'`（以及其他用到的类的 import）
> 2. `extends Phaser.Scene` 改为 `extends Scene`（用 import 进来的类，不用全局 `Phaser` 对象）
> 3. `startButton` 属性加上类型声明（`private startButton: Sprite`）
> 4. `create()` 方法加上返回类型（`create(): void`）
>
> `create()` 里面的游戏逻辑（`this.add.sprite`、`setInteractive()`、`on('pointerdown')`、`this.scene.start()`）**不需要改动**。

---

**Q3.** 为什么说"从 Phaser 3 迁移到 Phaser 4 不需要重新学游戏开发"？

> **答：** Phaser 4 改变的是工程层面的写法：import 方式、TypeScript 类型声明、模块路径。但所有的游戏开发核心概念 — Scene 生命周期、Sprite、Physics Body、Collision、Tween、Input — 完全没有变化，API 也高度相似。迁移时你只需要适应"怎么引入代码"的方式，游戏逻辑本身几乎不需要重写。
