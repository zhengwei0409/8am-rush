# 6.1 — Why Physics Engines

## 大局观 (High-Level Map)

到目前为止，你学的 Sprite 是"死的" — 你把它放在哪里它就在哪里，不会自己动，不会掉落，碰到墙壁也不会停下来。

真实的游戏里，物体需要有**重力**、**速度**、**碰撞**。你可以自己在 `update()` 里手动写这些逻辑，但那会非常复杂。

**Physics Engine（物理引擎）** 就是替你处理这一切的系统。

## 1. 没有物理引擎，游戏需要手动处理什么？

假设你要做一个简单的平台跳跃游戏，角色需要：

- 受重力影响，不断下落
- 跳起来后有抛物线轨迹
- 落到地板上停下来
- 碰到墙壁不能穿过去
- 被敌人碰到受伤

如果全部手动写，你需要：

```javascript
update(time, delta) {
    // 手动模拟重力
    this.velocityY += 9.8 * (delta / 1000);
    player.y += this.velocityY;

    // 手动检测地板碰撞
    if (player.y + player.height / 2 >= floor.y) {
        player.y = floor.y - player.height / 2;
        this.velocityY = 0;
    }

    // 手动检测墙壁碰撞（左、右各一次）
    // 手动检测每一个平台...
    // 手动检测每一个敌人...
}
```

场景越复杂，这段代码越长越容易出 bug。物理引擎把这些全部封装好了。

## 2. 什么是 Physics Engine（物理引擎）？

**Physics Engine** 是一个模拟物理规律的系统，它自动处理：

- 物体在力（重力、推力）下的运动
- 物体之间的碰撞和反弹
- 物体的速度和加速度变化

你不需要自己写物理公式，只需要告诉引擎"这个物体有多重"、"重力是多少"、"这两个物体能不能穿过对方"，引擎自动在每帧更新所有物体的位置和状态。

---

## 3. Velocity（速度）和 Acceleration（加速度）

这是物理引擎最基础的两个概念：

**Velocity（速度）** — 物体每秒移动多少像素，以及往哪个方向移动。

```
velocityX = 200   → 每秒向右移动 200 像素
velocityY = -300  → 每秒向上移动 300 像素（Y 轴向下为正，所以负数向上）
```

**Acceleration（加速度）** — 速度的变化率，即速度每秒增加/减少多少。

重力就是一种加速度：它让 `velocityY` 每秒不断增大，物体越落越快。

```
没有物理引擎：你要每帧手动计算 velocity += acceleration * deltaTime
有物理引擎：你只需设置 gravity: 300，引擎自动处理
```

---

## 4. Collision Detection（碰撞检测）

**Collision Detection** 是检测两个物体是否"接触"或"重叠"的过程。

碰撞检测在游戏里无处不在：
- 角色站在地板上（不穿透）
- 子弹打中敌人
- 玩家捡起道具
- 角色碰到边界墙壁

手动写碰撞检测需要比较每对物体的坐标和大小，场景里有 100 个物体就要比较 100×99/2 = 4950 对，性能会崩。

物理引擎用优化过的算法（空间分区等）高效地处理大量物体之间的碰撞检测。

---

## 5. Automated Updates（自动更新）

物理引擎最大的好处是**自动更新**。

每一帧，你不需要手动计算物体的新位置。物理引擎在 `update()` 执行之前，已经根据速度、加速度、碰撞结果，把所有物体的位置更新好了。

```
每帧的执行顺序：

[ 物理引擎更新所有物体位置 ]
           ↓
[ 你的 update() 代码执行 ]
           ↓
[ 渲染到屏幕 ]
```

你在 `update()` 里只需要处理"改变速度的指令"（例如按左键设置 velocityX = -200），物理引擎负责把速度转换成位置变化。

## 6. Realism vs Arcade Feel

物理引擎的模拟可以偏向两个方向：

**Realism（真实感）**
- 完整的物理模拟：摩擦力、转动惯量、复杂碰撞形状
- 物体行为接近现实世界
- 计算成本高
- 适合：模拟类游戏、物理谜题游戏

**Arcade Feel（街机感）**
- 简化的物理：只有速度、重力、矩形碰撞
- 物体行为简单可预测，容易控制
- 计算成本低
- 适合：平台跳跃、射击、大多数 2D 动作游戏

Phaser 提供两种物理系统对应这两个方向，6.2 和 6.3 会分别介绍。

## Self-Check

**Q1.** 物理引擎主要替开发者自动处理哪几件事？

> **答：**
> 1. **Velocity 和 Acceleration** — 自动根据速度和加速度更新物体位置
> 2. **Collision Detection** — 自动检测物体之间是否发生碰撞
> 3. **Automated Updates** — 每帧自动更新所有物体的物理状态，不需要手动计算

---

**Q2.** Velocity 和 Acceleration 有什么区别？

> **答：** Velocity（速度）是物体当前每秒移动多少像素、往哪个方向。Acceleration（加速度）是速度的变化率，即速度每秒增加或减少多少。重力是加速度的典型例子：它让 velocityY 每秒持续增大，使物体越落越快。

---

**Q3.** 为什么大多数 2D 动作游戏选择"Arcade Feel"的物理风格，而不是追求完整的物理真实感？

> **答：** Arcade 风格的物理（简单速度、矩形碰撞、固定重力）计算成本低，物体行为简单可预测，玩家容易掌控角色。完整的真实物理模拟虽然更逼真，但计算开销大，而且过于真实的物理反而让角色"难以操控"，降低游戏体验。大多数 2D 动作游戏追求的是"好玩"，不是"真实"。
