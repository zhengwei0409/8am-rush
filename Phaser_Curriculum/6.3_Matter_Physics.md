# 6.3 — Matter.js Physics

## 大局观 (High-Level Map)

在 6.2 你学了 Arcade Physics — 轻量、快，但只支持矩形碰撞，物体不会旋转。

有些游戏需要更真实的物理：角色滚下斜坡、箱子碰撞后翻转、圆形球体弹跳……这些 Arcade Physics 做不到。

这时候需要 Phaser 的第二个物理系统：**Matter.js**。

```
大图：
Phaser 物理系统
  ├── Arcade Physics — 简单、快、矩形碰撞、无旋转
  └── Matter.js      ← 你现在在这里（复杂、慢、任意形状、有旋转）
```

这节的目标不是学会怎么用 Matter.js，而是理解**它是什么、什么时候选它**。

## 1. 什么是 Matter.js？

**Matter.js** 是一个独立的 JavaScript 物理引擎库，Phaser 把它集成进来作为第二个物理选项。

Matter.js 是一个**完整的 2D 刚体物理模拟器**，目标是尽可能真实地模拟物理世界。

在 Config 里启用：
```javascript
physics: {
    default: 'matter',
    matter: {
        gravity: { y: 1 },
        debug: false
    }
}
```

注意：Arcade 和 Matter 的重力单位不同：
- Arcade：`gravity: { y: 300 }` 单位是像素/秒²
- Matter：`gravity: { y: 1 }` 是比例系数，1 = 正常重力

---

## 2. Rigid Body Dynamics（刚体动力学）

**Rigid Body（刚体）** 是物理学里的一个概念：一个**形状不变**的固体物体。

与之相对的是软体（Soft Body，如布料、果冻），它可以形变。Matter.js 只处理刚体。

**Dynamics（动力学）** 是研究力和运动关系的物理分支：力如何改变物体的速度、位置、旋转。

Matter.js 对每个刚体模拟：
- 质量（mass）— 越重越难推动
- 摩擦力（friction）— 物体滑动时的阻力
- 弹性（restitution）— 碰撞后反弹的程度（0 = 不弹，1 = 完全弹回）
- 惯性（inertia）— 旋转时的阻力

这些属性让物体的行为接近真实物理。

---

## 3. Rotational Physics（旋转物理）

这是 Matter.js 和 Arcade Physics 最明显的区别。

**Arcade Physics：** 物体永远不会因为碰撞而旋转。一个正方形箱子撞到斜坡，它会停下来，但不会翻转。

**Matter.js：** 物体碰撞后会根据碰撞角度和力矩产生**真实的旋转**。

```
Arcade Physics：
  [箱子] → 碰到斜坡 → [箱子] 停在斜坡上（没有旋转）

Matter.js：
  [箱子] → 碰到斜坡 → [箱子] 翻滚下去（真实旋转）
```

旋转物理需要计算**力矩（torque）** — 力施加在物体不同位置时产生的旋转效果。这是 Matter.js 计算量远大于 Arcade 的主要原因之一。

---

## 4. Complex Shapes（复杂碰撞形状）

Arcade Physics 只支持矩形（AABB）。

Matter.js 支持：
- **Circle（圆形）** — 球、硬币、眼球
- **Polygon（多边形）** — 三角形、六边形、任意凸多边形
- **Compound Body（复合体）** — 把多个形状拼在一起，组成一个复杂轮廓

```
Arcade：   □ □ □ □   (只能用矩形)

Matter：   □ ○ △ ⬡   (矩形、圆形、三角形、六边形……)
```

复杂形状让碰撞更精确 — 一个圆形角色用圆形碰撞框，不会在矩形框的角落卡住地形。

---

## 5. 性能代价（Higher Cost）

Matter.js 的真实感是有代价的：

| 对比项 | Arcade Physics | Matter.js |
|--------|---------------|-----------|
| 碰撞检测算法 | AABB（极简） | SAT / GJK（复杂） |
| 旋转计算 | 无 | 每个物体每帧计算力矩 |
| 约束系统 | 无 | 支持关节、弹簧（额外计算） |
| 每帧可承受物体数量 | 数百个 | 数十个（复杂场景） |

简单来说：同样的硬件，Matter.js 能稳定运行的物理物体数量比 Arcade Physics 少得多。

---

## 6. 什么时候选 Matter，什么时候选 Arcade？

**选 Arcade Physics，如果：**
- 平台跳跃（Mario 风格）
- 射击游戏（子弹、敌人）
- 大多数 2D 动作游戏
- 场景里有很多物理物体（> 50 个）
- 不需要旋转或复杂形状

**选 Matter.js，如果：**
- 物体需要真实旋转（滚动的圆形、翻倒的箱子）
- 需要圆形或多边形碰撞（弹珠台、切水果）
- 需要关节/绳索（链条、摆动物体）
- 物理本身是游戏核心玩法（物理解谜）

> **经验法则：** 先考虑 Arcade。只有当 Arcade 明确无法实现你需要的效果时，才切换到 Matter.js。

## Self-Check

**Q1.** Matter.js 相比 Arcade Physics，多了哪两个 Arcade 完全没有的物理特性？

> **答：**
> 1. **Rotational Physics（旋转物理）** — 物体碰撞后会根据力矩产生真实旋转，Arcade 里物体永远不会因碰撞旋转。
> 2. **Complex Shapes（复杂碰撞形状）** — Matter.js 支持圆形、多边形、复合体，Arcade 只支持矩形。

---

**Q2.** 为什么 Matter.js 能模拟的物理物体数量比 Arcade Physics 少？

> **答：** Matter.js 使用更复杂的碰撞检测算法（SAT/GJK，支持任意形状），还需要每帧为每个物体计算旋转力矩、摩擦力、弹性等参数。这些计算量远大于 Arcade 的 AABB 矩形检测，所以同样的硬件，Matter.js 能稳定处理的物体数量要少得多。

---

**Q3.** 你正在做一个平台跳跃游戏（类似 Super Mario），应该选 Arcade Physics 还是 Matter.js？说明理由。

> **答：** 应该选 **Arcade Physics**。平台跳跃游戏的物理需求是：角色受重力下落、站在平台上、左右移动、跳跃。这些都是简单的矩形碰撞 + 速度控制，Arcade Physics 完全够用。Matter.js 的额外特性（旋转、复杂形状）对这类游戏没有帮助，反而会增加性能开销。
