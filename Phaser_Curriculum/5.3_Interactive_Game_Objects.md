# 5.3 — Interactive Game Objects

## 大局观 (High-Level Map)

在 5.2 你学了怎么追踪 Pointer 的位置和点击。但那是"检测玩家点击了屏幕哪个位置"。

实际游戏里，你更想知道的是"玩家点击了**哪个游戏对象**"— 比如点击了哪个按钮、哪个敌人、哪个道具。

问题是：Sprite 默认是不可点击的，Phaser 不会自动帮你检测"点击是否落在某个 Sprite 上"。

你需要明确地告诉 Phaser："这个 Game Object 可以被点击"，这就是 `setInteractive()` 的作用。

```

```

## 1. 为什么 Sprite 默认不可点击？

Phaser 的设计原则是"**按需开启**"。

一个游戏场景可能有几百个 Sprite — 背景树木、地板砖块、装饰物……这些都不需要被点击。如果 Phaser 对每一个 Sprite 都进行"点击检测"，每帧要做大量无用的计算，白白浪费性能。

所以 Phaser 默认**关闭**所有 Game Object 的点击检测。只有当你调用 `setInteractive()`，Phaser 才开始追踪那个对象的点击状态。

---

## 2. setInteractive() — 启用输入

**`setInteractive()`** 是让一个 Game Object 开始响应输入事件的开关。

```javascript
// 在 create() 里
const button = this.add.sprite(400, 300, 'button');
button.setInteractive();
```

调用之后，Phaser 就会在每帧检测：Pointer 是否在这个 Sprite 的范围内？是否按下了？

`setInteractive()` 也可以链式调用：
```javascript
const button = this.add.sprite(400, 300, 'button').setInteractive();
```

---

## 3. Hit Area（点击区域）是什么？

**Hit Area** 是决定 Sprite 的哪个范围内的点击会被识别的区域。

默认情况下，调用 `setInteractive()` 后，Hit Area 是 Sprite 的**矩形边界框（bounding box）** — 即图片的宽高范围。

```
┌─────────────────┐
│                 │
│    [图片内容]    │  ← 点击这个矩形内任何地方都算点击
│                 │
└─────────────────┘
   ↑ Hit Area（默认 = 图片矩形）
```

对于矩形按钮这完全够用。但如果图片是一个圆形图标，玩家点击圆角外的空白区域也会触发，感觉不对。

进阶：可以传入自定义形状，但这超出本节范围（课程标注"Not required: custom hit shapes"）。

## 4. Pointer Events — 监听对象上的点击

启用 `setInteractive()` 之后，你就可以在这个 Game Object 上监听各种 Pointer 事件：

```javascript
const button = this.add.sprite(400, 300, 'button').setInteractive();

// 点击（按下）
button.on('pointerdown', () => {
    console.log('按钮被按下了');
});

// 松开
button.on('pointerup', () => {
    console.log('按钮松开了');
});

// Pointer 移入这个 Sprite 的范围
button.on('pointerover', () => {
    button.setTint(0xffff00);  // 变黄色，做 hover 效果
});

// Pointer 移出这个 Sprite 的范围
button.on('pointerout', () => {
    button.clearTint();  // 恢复原色
});
```

常用的 Pointer 事件：

| 事件名 | 触发时机 |
|--------|---------|
| `'pointerdown'` | Pointer 在这个对象上按下 |
| `'pointerup'` | Pointer 在这个对象上松开 |
| `'pointerover'` | Pointer 进入这个对象的范围（hover） |
| `'pointerout'` | Pointer 离开这个对象的范围 |

## 5. Event Listeners 是什么？

**Event Listener（事件监听器）** 是"当某件事发生时，执行这段代码"的机制。

你用 `.on(事件名, 回调函数)` 来注册一个 Listener：

```javascript
button.on('pointerdown', () => {
    // 这个函数 = Event Listener
    // 当 'pointerdown' 事件发生时，Phaser 会自动调用这个函数
    startGame();
});
```

**关键理解：**
- 你不需要在 `update()` 里每帧检查按钮有没有被点击
- 你只需要在 `create()` 里注册一次 Listener
- Phaser 内部在检测到点击时，会自动调用你的回调函数

这和 5.1 里键盘的 polling 模式不同 — 这是**事件驱动（Event-Driven）**的模式。

**Polling vs Event-Driven：**

| | Polling（5.1 键盘） | Event-Driven（5.3 对象点击） |
|---|---|---|
| 检查方式 | 你每帧主动查询 | Phaser 检测到后通知你 |
| 代码位置 | `update()` 里 | `create()` 里注册一次 |
| 适合场景 | 持续性输入（持续按住移动） | 一次性触发（按钮点击） |

## 6. Object-Level Input vs Scene-Level Input

这是 5.2 和 5.3 的核心区别：

**Scene-Level Input（5.2）：**
```javascript
// 整个场景的点击
this.input.on('pointerdown', (pointer) => {
    console.log('场景里某个地方被点击了，位置：', pointer.x, pointer.y);
});
```
你只知道"屏幕哪个坐标被点击了"，自己判断是否命中某个对象。

**Object-Level Input（5.3）：**
```javascript
// 具体对象的点击
button.setInteractive();
button.on('pointerdown', () => {
    console.log('这个按钮被点击了');
});
```
Phaser 帮你做了"点击是否在这个对象上"的判断，你直接知道"哪个对象被点击了"。

> **实际开发中：** 菜单按钮、道具图标、可交互 NPC 用 Object-Level Input。检测"点击空地"、"点击地图任意位置"用 Scene-Level Input。

## Self-Check

**Q1.** 为什么 Phaser 的 Game Object 默认不响应点击，需要调用 `setInteractive()` 才能启用？

> **答：** 出于性能考虑。一个场景可能有几百个 Game Object，如果每个都自动进行点击检测，每帧要做大量无用计算。Phaser 采用"按需开启"的设计：只有你明确调用 `setInteractive()` 的对象，Phaser 才会追踪它的点击状态，其他对象不参与计算。

---

**Q2.** 下面这段代码想在玩家点击 `enemy` Sprite 时把它销毁，但运行后点击没有反应，为什么？

```javascript
const enemy = this.add.sprite(200, 200, 'enemy');

enemy.on('pointerdown', () => {
    enemy.destroy();
});
```

> **答：** 缺少 `enemy.setInteractive()`。没有调用 `setInteractive()`，Phaser 不会对这个 Sprite 进行点击检测，所以 `'pointerdown'` 事件永远不会触发。修复方法：在注册事件之前加上 `enemy.setInteractive()`。

---

**Q3.** `pointerover` 和 `pointerdown` 有什么区别？各适合做什么效果？

> **答：**
> - `pointerover`：Pointer **移入**对象范围时触发，不需要点击。适合做 **hover 效果**，例如鼠标悬停时按钮变色、放大，给玩家视觉反馈"这个东西可以点击"。
> - `pointerdown`：Pointer **在对象上按下**时触发。适合做**实际的点击动作**，例如点击按钮开始游戏、点击道具拾取。
