## 1. 什么是 Arcade Physics？

**Arcade Physics** 是 Phaser 内置的轻量物理系统，名字来源于"街机游戏（Arcade Game）"风格。

它的设计哲学是：**简单、快速、够用**。

它提供的功能：
- 给物体设置速度（velocity）
- 全局重力（gravity）
- 矩形碰撞检测（AABB）
- 静态物体和动态物体

它**不提供**的功能：
- 旋转惯量（物体碰撞后不会旋转）
- 复杂碰撞形状（圆形、多边形）
- 软体、流体模拟

90% 的 2D 游戏（平台跳跃、射击、益智）用 Arcade Physics 完全够用。

---

## 2. 怎么启用 Arcade Physics？

在游戏配置（Config）里声明使用 Arcade Physics：

```javascript
const config = {
    type: Phaser.AUTO,
    width: 800,
    height: 600,
    physics: {
        default: 'arcade',       // 使用 Arcade Physics
        arcade: {
            gravity: { y: 300 }, // 全局重力（每秒向下加速 300 像素）
            debug: false         // 是否显示碰撞框（调试用）
        }
    },
    scene: [GameScene]
};
```

启用后，在 Scene 里通过 `this.physics` 访问物理系统。

让一个 Sprite 具有物理属性，用 `this.physics.add.sprite()` 代替 `this.add.sprite()`：

```javascript
// 普通 Sprite（没有物理）
const sprite = this.add.sprite(100, 100, 'player');

// 带物理的 Sprite
const player = this.physics.add.sprite(100, 100, 'player');
```

区别：`this.physics.add.sprite()` 创建的 Sprite 会自动附带一个 **Physics Body**（6.4 会详细讲），物理引擎通过 Body 来控制这个 Sprite 的运动。

## 3. AABB Collision（矩形碰撞）

**AABB** = **Axis-Aligned Bounding Box**，直译是"轴对齐边界框"。

简单来说，就是用一个**不旋转的矩形**来代表物体的碰撞范围。

```
┌─────────────────┐
│                 │
│  [Sprite 图片]  │  ← AABB（矩形边界框）
│                 │
└─────────────────┘
```

两个物体碰撞检测的逻辑：
```
如果 A 的矩形 和 B 的矩形 有重叠部分 → 发生碰撞
```

这个计算非常简单，只需比较四个坐标（左、右、上、下），CPU 负担极小。

**AABB 的限制：**
- 物体只能用矩形，不能用圆形或多边形
- 物体旋转后，碰撞框不跟着旋转（始终保持水平/垂直方向）

对于大多数 2D 游戏角色（走路、跳跃、射击），矩形碰撞框已经完全足够，玩家感知不到这点近似误差。

## 4. Gravity（重力）

**Gravity** 是一个向下的持续加速度，让物体自然下落。

在 Config 里设置全局重力：
```javascript
arcade: {
    gravity: { y: 300 }
}
```

`gravity: { y: 300 }` 意思是：每秒给所有动态物体的 `velocityY` 增加 300。

效果：
- 一个初始静止的物体，1 秒后 velocityY = 300（向下移动 300px/s）
- 2 秒后 velocityY = 600（越落越快）
- 直到碰到地板，碰撞阻止它继续下落

也可以给单个物体设置不同重力，或者关闭某个物体的重力：
```javascript
player.body.setGravityY(500);     // 这个物体重力更大
platform.body.setAllowGravity(false);  // 平台不受重力影响（静态物体）
```

## 5. Velocity-based Motion（基于速度的移动）

Arcade Physics 里，移动物体的标准方式是**设置速度（velocity）**，而不是直接改变坐标。

```javascript
// 错误做法（绕过物理引擎）
player.x += 5;  // 直接改坐标，物理引擎不知道你做了什么

// 正确做法（通过物理引擎）
player.body.setVelocityX(200);  // 告诉物理引擎：这个物体往右以 200px/s 移动
```

为什么要用 velocity 而不是直接改坐标？

因为碰撞检测是在**速度层面**工作的。物理引擎根据速度预测物体下一帧的位置，然后在到达那个位置之前检测是否会碰到其他物体。如果直接改坐标，物体可能"瞬移"穿过薄墙，碰撞检测失效。

在 `update()` 里控制移动的标准写法：
```javascript
update() {
    if (this.cursors.left.isDown) {
        player.body.setVelocityX(-200);   // 向左
    } else if (this.cursors.right.isDown) {
        player.body.setVelocityX(200);    // 向右
    } else {
        player.body.setVelocityX(0);      // 停止
    }

    if (this.cursors.up.isDown && player.body.blocked.down) {
        player.body.setVelocityY(-400);   // 跳跃（只在站在地面时）
    }
}
```

## 6. 性能特点

Arcade Physics 之所以"轻量"，是因为几个刻意的设计取舍：

| 特点 | 说明 |
|------|------|
| AABB 碰撞 | 数学上最简单的碰撞检测，计算量极小 |
| 无旋转物理 | 不计算旋转惯量，省去大量计算 |
| 离散碰撞检测 | 每帧检测一次，而非连续检测（高速物体可能穿墙，但一般游戏感知不到） |

结论：Arcade Physics 可以轻松支持数百个物理物体同时在场景里运动，适合绝大多数 2D 游戏。

## Self-Check

**Q1.** 什么是 AABB？Arcade Physics 用 AABB 做碰撞检测有什么限制？

> **答：** AABB（Axis-Aligned Bounding Box）是一个不旋转的矩形边界框，用来代表物体的碰撞范围。碰撞检测通过比较两个矩形是否重叠来判断是否碰撞，计算简单高效。限制是：只能用矩形，不支持圆形或多边形；物体旋转时碰撞框不跟着旋转，始终保持水平/垂直方向。

---

**Q2.** 为什么移动物体应该用 `setVelocityX()` 而不是直接修改 `player.x`？

> **答：** 碰撞检测在速度层面工作。物理引擎根据速度预测下一帧位置，在"到达"之前检测是否会碰撞。直接改 `player.x` 会让物体"瞬移"，物理引擎无法正确检测碰撞，可能导致物体穿过墙壁等 bug。

---

**Q3.** Config 里设置 `gravity: { y: 300 }` 对场景里的物体有什么具体效果？

> **答：** 场景里所有动态物理物体每秒的 `velocityY` 会增加 300，使物体持续向下加速，模拟重力。一个初始静止的物体，1 秒后向下速度为 300px/s，2 秒后为 600px/s，越落越快，直到被地板或其他物体的碰撞阻止。
