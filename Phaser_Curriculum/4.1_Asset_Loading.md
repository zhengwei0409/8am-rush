# 4.1 — Asset Loading

## 大局观 (High-Level Map)

在游戏开发的整个流程里，**Asset Loading** 属于"资源管理"这一层。
在你能在屏幕上看到任何图片或听到任何声音之前，游戏引擎必须先把这些文件从硬盘（或服务器）加载到内存里。
这是 Section 4 的第一步，因为后面所有的 Sprites、Animations、Text 都依赖于已经加载好的资源。

```
大图：
[ 游戏启动 ]
     ↓
[ preload() — 加载资源 ]  ← 你现在在这里
     ↓
[ create() — 使用资源 ]
     ↓
[ update() — 每帧运行 ]
```

## 1. 什么是 Asset（资源）？

**Asset** 就是游戏用到的所有外部文件，包括：

- 图片（`.png`, `.jpg`）— 角色、背景、图标
- 音频（`.mp3`, `.ogg`）— 背景音乐、音效
- 数据文件（`.json`）— 地图、配置

这些文件不是写在代码里的，而是**独立存放**在 `assets/` 文件夹里的外部文件。

---

## 2. 为什么需要"加载"这个步骤？

浏览器里运行的 JavaScript **不能直接读取本地文件**，它需要通过网络请求（HTTP request）去获取每一个文件。

网络请求是**异步**的（async）— 意思是你发出请求之后，不知道文件什么时候才到。如果你还没等文件到就开始使用它，程序会崩溃。

所以 Phaser 设计了一个专门的加载阶段，在游戏真正开始之前，**先把所有需要的文件全部下载并放进内存**，确保后面用的时候一切都准备好了。

---

## 3. Phaser 的 Loader System

**Loader System** 是 Phaser 内置的资源管理模块。它负责：

1. 接收你列出的所有资源请求
2. 并行（同时）发送网络请求去下载它们
3. 追踪下载进度
4. 全部完成后，通知 Phaser 可以进入下一阶段（`create()`）

你不需要手动写网络请求代码。Phaser 把这一切都封装好了，你只需要**告诉它要加载什么**。

---

## 4. Asset Key（资源键）

**Asset Key** 是你给每个资源取的一个**唯一名字（字符串）**。

加载的时候，你给资源取名：
```
this.load.image('player', 'assets/player.png');
                 ↑ key      ↑ 文件路径
```

使用的时候，你用这个名字来引用资源：
```
this.add.sprite(100, 100, 'player');
                           ↑ 用同一个 key
```

Key 就像一个**标签**，把"文件"和"代码里的引用"连接起来。

**重要规则：**
- Key 必须唯一 — 两个资源不能用同一个 key
- Key 是大小写敏感的（`'Player'` ≠ `'player'`）
- Key 是普通字符串，你自己决定叫什么

---

## 5. 怎么加载 Image 和 Audio

### 加载 Image（图片）

```javascript
this.load.image('sky', 'assets/sky.png');
```

- 第一个参数：key（你取的名字）
- 第二个参数：文件路径（相对于项目根目录）

### 加载 Audio（音频）

```javascript
this.load.audio('bgm', 'assets/music.mp3');
```

同样的结构：key + 文件路径。

Phaser 的 Loader 提供了很多 `load.xxx()` 方法，分别对应不同文件类型，但**格式都一样**：key + path。

---

## 6. 为什么必须在 `preload()` 里加载？

Phaser 的 Scene 有固定的执行顺序：

```
preload()  →  create()  →  update() → update() → update() ...
```

**`preload()`** 是唯一一个 Phaser 会**等待所有加载完成才继续**的方法。

- 如果你在 `preload()` 里调用 `this.load.image()`，Phaser 会等所有文件下载完毕，然后才进入 `create()`。
- 如果你在 `create()` 或 `update()` 里加载，Phaser 不会等，你的资源可能还没到，代码就已经尝试使用它了，结果报错。

> **关键原则：** 所有需要用到的资源，都在 `preload()` 里声明，Phaser 负责确保它们全部准备好。

## Self-Check

**Q1.** Asset Key 的作用是什么？为什么两个资源不能用同一个 key？

> **答：** Asset Key 是资源的唯一标识符（字符串名字），用来在代码里引用已加载的资源。两个资源不能用同一个 key，是因为 key 是 Loader 查找资源的唯一依据 — 如果重复，Phaser 不知道你想用哪一个。

---

**Q2.** 下面这段代码有什么问题？

```javascript
create() {
    this.load.image('hero', 'assets/hero.png');
    this.add.sprite(200, 200, 'hero');
}
```

> **答：** `this.load.image()` 应该放在 `preload()` 里，不应该放在 `create()` 里。在 `create()` 里加载资源，Phaser 不会等待下载完成就继续执行下一行，所以 `this.add.sprite()` 运行时 `'hero'` 还没准备好，会报错或显示空白。

---

**Q3.** Phaser 的 Loader System 主要负责做什么事？（列出 3 点）

> **答：**
> 1. 接收你在 `preload()` 里列出的所有资源请求
> 2. 并行发送网络请求去下载这些文件
> 3. 等所有文件下载完毕后，才通知 Phaser 进入 `create()` 阶段
