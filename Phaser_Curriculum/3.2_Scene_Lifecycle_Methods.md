# 3.2 — Scene Lifecycle Methods

## 大局观 (High-Level Map)

上一节说每个 Scene 都有自己的 Lifecycle（生命周期），包含几个阶段。

这一节我们聚焦在最核心的三个阶段对应的方法：`preload()`、`create()`、`update()`。

这三个方法是你在 Phaser 里写游戏逻辑的主战场。理解它们各自的职责和执行顺序，是写好 Phaser 游戏的基础。

## 1. 什么是 Lifecycle Method？

**Lifecycle Method（生命周期方法）** 是引擎在特定时机自动调用的函数。

你不需要自己调用这些函数——Phaser 在正确的时机会自动执行它们。你只需要在函数体里填入你想在那个时机执行的代码。

就像你定了一个闹钟，时间到了闹钟自己响——你不需要手动触发它，只需要提前设置好"响的时候做什么"。

---

## 2. 三个核心方法

### 2.1 preload() — 预加载资源

```javascript
preload() {
    this.load.image("player", "assets/images/player.png");
    this.load.audio("jump", "assets/audio/jump.mp3");
}
```

**执行时机：** Scene 启动后，`create()` 之前，**只执行一次**。

**职责：** 告诉 Phaser 这个 Scene 需要用到哪些外部资源（图片、音效等），让 Phaser 提前把它们下载到内存里。

**为什么要单独有一个 preload？**

加载文件是**异步**的——浏览器需要时间从服务器下载文件，下载期间游戏不能继续。`preload()` 是一个专门的阶段，Phaser 会等所有资源加载完毕后，才进入 `create()`。

**Asynchronous（异步）** 的意思是：某个操作不会立刻完成，需要等待，但程序不会停下来干等——它可以先做别的事，等操作完成后再继续。Phaser 的 Loader 帮你管理了这个等待过程。

---

### 2.2 create() — 初始化场景

```javascript
create() {
    this.player = this.add.sprite(400, 300, "player");
    this.score = 0;
    this.cursors = this.input.keyboard.createCursorKeys();
}
```

**执行时机：** `preload()` 完成后，**只执行一次**。

**职责：** 设置这个 Scene 需要的所有东西：
- 创建游戏对象（角色、背景、UI）
- 初始化变量（分数、血量）
- 设置输入监听
- 配置物理和碰撞规则

`create()` 是"房间布置"阶段——在游戏开始之前，把所有东西摆放好。

执行完 `create()` 之后，游戏才正式进入运行状态，开始执行 `update()`。

---

### 2.3 update() — 每帧更新

```javascript
update() {
    if (this.cursors.left.isDown) {
        this.player.x -= 5;
    }
    if (this.cursors.right.isDown) {
        this.player.x += 5;
    }
}
```

**执行时机：** `create()` 完成后，**每帧执行一次**，直到 Scene 停止。

**职责：** 处理每帧需要更新的游戏逻辑：
- 读取输入并移动角色
- 更新敌人 AI
- 检查游戏条件（玩家是否死亡、是否过关）
- 更新 UI 显示

`update()` 就是 1.3 节游戏循环里的 **Update 阶段**——它在每一帧都会被 Phaser 调用，是游戏持续运行的核心。

---

## 3. 执行顺序图

```
Scene 被启动
    ↓
preload()    ← 执行一次：加载资源，等待完成
    ↓
create()     ← 执行一次：初始化对象和逻辑
    ↓
┌──────────┐
│ update() │ ← 每帧执行：处理输入、更新状态
│ update() │
│ update() │
│  ...     │
└──────────┘
    ↓
Scene 停止
```

顺序是固定的，不可改变：preload → create → update（循环）。

---

## 4. Per-frame vs One-time

这是理解三个方法的关键区分：

| 方法 | 执行频率 | 类型 |
|------|----------|------|
| `preload()` | 一次 | One-time（一次性） |
| `create()` | 一次 | One-time（一次性） |
| `update()` | 每帧一次 | Per-frame（逐帧） |

**One-time（一次性）** 方法：在 Scene 的生命周期里只执行一次，做的是"准备工作"。

**Per-frame（逐帧）** 方法：每帧都执行，做的是"持续运作的逻辑"。

常见错误：把应该在 `create()` 里做的事放进 `update()` 里。

例如，在 `update()` 里每帧创建一个新角色 → 60 FPS 的游戏每秒会创建 60 个角色，游戏立刻崩溃。创建角色应该在 `create()` 里做，只做一次。

## Self-Check

**Q1.** 什么是 Lifecycle Method？Phaser 里为什么有这个概念？

> **答：** Lifecycle Method 是引擎在特定时机自动调用的函数。因为游戏的不同阶段（加载资源、初始化、持续运行）需要在正确的时机执行不同的代码，Phaser 定义好这些时机对应的方法，开发者只需在里面填写逻辑，引擎负责在正确时机调用。

---

**Q2.** `preload()` 的职责是什么？为什么资源加载要在单独的阶段进行？

> **答：** `preload()` 负责告诉 Phaser 需要加载哪些外部资源（图片、音效等）。资源加载是异步的，需要时间等待下载完成。Phaser 会在 `preload()` 阶段等所有资源加载完毕后，才进入 `create()`，确保游戏对象创建时资源已经就绪。

---

**Q3.** `create()` 和 `update()` 最大的区别是什么？

> **答：** `create()` 是 One-time（一次性）的，在 Scene 启动时只执行一次，负责初始化游戏对象和逻辑。`update()` 是 Per-frame（逐帧）的，每帧执行一次，负责处理持续运行的游戏逻辑（输入、移动、AI等）。

---

**Q4.** 三个方法的执行顺序是什么？

> **答：** `preload()` → `create()` → `update()`（每帧循环）。顺序固定不变。

---

**Q5.** 下面这段逻辑放在哪个方法里更合适？为什么？

> "在坐标 (100, 200) 创建一个玩家角色"

> **答：** 应该放在 `create()` 里。创建游戏对象是"初始化"工作，只需要做一次。如果放在 `update()` 里，每帧都会创建一个新角色，导致游戏出错。
