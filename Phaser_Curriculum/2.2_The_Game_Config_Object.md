# 2.2 — The Game Config Object

## 大局观 (High-Level Map)

你已经知道 Phaser 项目的文件结构了。现在问题是：游戏是怎么"启动"的？

答案是：你需要创建一个 **Phaser.Game** 对象，而创建它的第一步，就是提供一个 **Game Config Object（游戏配置对象）**。

Config 就是游戏的"出生设定"——在游戏跑起来之前，你告诉 Phaser：这个游戏有多大？用什么渲染器？有哪些场景？

---

## 1. 什么是 Config Object？

在 JavaScript 里，**Object（对象）** 是一种用 `{}` 包裹、存放多个键值对的数据结构：

```javascript
const person = {
    name: "Wei",
    age: 22
};
```

**Game Config Object** 就是一个普通的 JavaScript Object，里面放的是 Phaser 游戏的各种配置项。你把这个 Object 传给 `new Phaser.Game(config)`，Phaser 就会按照你的设定初始化整个游戏。

```javascript
const config = {
    // 配置项放在这里
};

const game = new Phaser.Game(config);
```

---

## 2. 五个核心配置字段

### 2.1 width 和 height — 游戏画布大小

```javascript
const config = {
    width: 800,
    height: 600
};
```

这两个字段定义了游戏 **Canvas（画布）** 的尺寸，单位是 **Pixel（像素）**。

- `width: 800` → 游戏画面宽 800 像素
- `height: 600` → 游戏画面高 600 像素

这决定了游戏的可见区域大小。画布外面的东西，玩家是看不到的（除非你用到 Camera，那是后面的话题）。

---

### 2.2 type — 渲染器类型

```javascript
const config = {
    type: Phaser.AUTO
};
```

`type` 告诉 Phaser 使用哪种渲染方式。有三个选项：

| 值 | 意思 |
|----|------|
| `Phaser.AUTO` | 自动选择：优先 WebGL，不支持则用 Canvas |
| `Phaser.WEBGL` | 强制使用 WebGL |
| `Phaser.CANVAS` | 强制使用 Canvas |

实际开发中，几乎总是用 `Phaser.AUTO`。WebGL 性能更好，但极少数旧设备不支持，AUTO 保证兼容性。

渲染器的细节会在 2.3 节详细讲，这里只需要知道这个字段存在和它的三个选项。

---

### 2.3 parent — 父级 DOM 元素

```javascript
const config = {
    parent: "game-container"
};
```

**DOM（Document Object Model）** 是浏览器把 HTML 文档表示成对象树的方式，每一个 HTML 标签都是一个 DOM 元素。

`parent` 字段告诉 Phaser：把游戏的 `<canvas>` 元素放在哪个 HTML 元素里面。

例如你的 HTML 里有：
```html
<div id="game-container"></div>
```

设置 `parent: "game-container"` 后，Phaser 会自动在这个 `<div>` 内部创建 `<canvas>`，游戏画面就会显示在那里。

如果不设置 `parent`，Phaser 默认把 canvas 直接放在 `<body>` 里。

---

### 2.4 scene — 场景列表

```javascript
const config = {
    scene: [MenuScene, GameScene, UIScene]
};
```

**Scene（场景）** 是游戏的独立状态单元（1.1 里提过，会在第 3 节详细讲）。

`scene` 字段接受一个 **Array（数组）**，列出游戏里所有的场景。

- 数组里**第一个**场景会在游戏启动时自动运行
- 其余场景待命，等需要时再启动

这个字段告诉 Phaser"这个游戏有哪些场景"，Phaser 会把它们都注册到 **Scene Manager（场景管理器）** 里。

---

### 2.5 完整结构回顾

| 字段 | 作用 | 常见值 |
|------|------|--------|
| `width` | 画布宽度（像素） | `800` |
| `height` | 画布高度（像素） | `600` |
| `type` | 渲染器类型 | `Phaser.AUTO` |
| `parent` | 放置 canvas 的 HTML 元素 ID | `"game-container"` |
| `scene` | 游戏的所有场景（数组） | `[MenuScene, GameScene]` |

---

## 3. 完整的 Config 例子

```javascript
const config = {
    type: Phaser.AUTO,
    width: 800,
    height: 600,
    parent: "game-container",
    scene: [BootScene, MenuScene, GameScene]
};

const game = new Phaser.Game(config);
```

这几行代码就启动了一个完整的 Phaser 游戏：
- 自动选择 WebGL 或 Canvas 渲染
- 画布大小 800 × 600 像素
- canvas 放在 id 为 `"game-container"` 的元素里
- 游戏有三个场景，`BootScene` 最先运行

## Self-Check

**Q1.** Game Config Object 是什么？它在 Phaser 里的作用是什么？

> **答：** Game Config Object 是一个 JavaScript 对象，包含游戏的初始配置（画布大小、渲染器、场景等）。把它传给 `new Phaser.Game(config)`，Phaser 就会按照这些设定初始化游戏。

---

**Q2.** `width` 和 `height` 字段定义的是什么？单位是什么？

> **答：** 定义游戏 Canvas（画布）的尺寸，单位是 Pixel（像素）。

---

**Q3.** `type: Phaser.AUTO` 的意思是什么？

> **答：** 让 Phaser 自动选择渲染器：优先使用 WebGL（性能更好），如果浏览器不支持 WebGL，则回退到 Canvas 渲染。

---

**Q4.** 如果 config 里设置了 `scene: [BootScene, MenuScene, GameScene]`，游戏启动时哪个场景最先运行？

> **答：** `BootScene`，因为它是数组里的第一个元素，Phaser 会在游戏启动时自动运行第一个场景。

---

**Q5.** `parent` 字段的作用是什么？如果不设置它会怎样？

> **答：** `parent` 告诉 Phaser 把游戏的 `<canvas>` 元素放在哪个 HTML 元素里（通过 ID 指定）。如果不设置，Phaser 默认把 canvas 直接放在 `<body>` 里。
