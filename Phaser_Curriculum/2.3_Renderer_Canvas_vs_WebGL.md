# 2.3 — Renderer: Canvas vs WebGL

## 大局观 (High-Level Map)

上一节我们在 Config 里看到了 `type: Phaser.AUTO`，提到 Phaser 有两种渲染方式：**Canvas** 和 **WebGL**。

这一节我们搞清楚：这两种渲染器到底有什么区别？各有什么优劣？什么时候用哪个？

---

## 1. 什么是 Renderer？

**Renderer（渲染器）** 是游戏引擎里负责"把游戏画到屏幕上"的那个模块。

每一帧，Renderer 的工作是：读取所有游戏对象的位置、图像等信息，然后把它们绘制到 Canvas 上，变成玩家看到的画面。

Phaser 提供两种不同的 Renderer，底层技术不同，性能和能力也不同。

---

## 2. Canvas Renderer

### 什么是 Canvas？

**Canvas** 是 HTML5 提供的一个元素：`<canvas>`。

它本质上是一块"空白画布"。浏览器提供了一套叫 **Canvas 2D API** 的接口，让 JavaScript 可以在上面画图形——矩形、圆形、图片、文字等等。

Canvas 2D API 是由 **CPU（Central Processing Unit，中央处理器）** 来执行的。CPU 是电脑的"大脑"，负责处理通用计算任务。

### Canvas Renderer 的特点

| 特点 | 说明 |
|------|------|
| **兼容性好** | 所有现代浏览器（甚至较旧的浏览器）都支持 Canvas |
| **CPU 执行** | 绘图由 CPU 完成，不依赖显卡 |
| **性能较低** | CPU 不擅长大量并行图形计算，对象多了会变慢 |
| **简单直接** | 适合对象数量少、画面简单的游戏 |

---

## 3. WebGL Renderer

### 什么是 WebGL？

**WebGL（Web Graphics Library）** 是另一套浏览器 API，它允许 JavaScript 直接调用 **GPU（Graphics Processing Unit，图形处理器）** 的能力。

GPU 是专门为图形计算设计的芯片，它有成千上万个小核心，可以同时并行处理大量像素——这正是游戏渲染需要的。

WebGL 让浏览器游戏获得了接近原生应用的图形性能。

### WebGL Renderer 的特点

| 特点 | 说明 |
|------|------|
| **性能强** | 由 GPU 执行，处理大量精灵、粒子效果时依然流畅 |
| **支持 Shader** | 可以编写特殊的视觉效果程序（着色器） |
| **兼容性稍差** | 极少数旧设备或旧浏览器不支持 |
| **更复杂** | 底层更复杂，但 Phaser 帮你封装好了 |

**Shader（着色器）** 是一段运行在 GPU 上的程序，可以实现光照、模糊、色调变换等视觉效果。Canvas 渲染器不支持 Shader。

---

## 4. AUTO 模式

```javascript
const config = {
    type: Phaser.AUTO
};
```

`Phaser.AUTO` 的行为：

```
游戏启动
    ↓
浏览器支持 WebGL？
    ├─ 是 → 使用 WebGL Renderer（性能更好）
    └─ 否 → 回退到 Canvas Renderer（兼容性保障）
```

**Fallback（回退）** 是指当首选方案不可用时，自动切换到备选方案。

AUTO 模式让你同时得到"尽可能好的性能"和"最大兼容性"，是实际开发中的默认选择。

---

## 5. 性能权衡（Performance Tradeoffs）

**Tradeoff（权衡）** 的意思是：选择 A 会得到某些好处，但同时放弃某些东西；选择 B 则相反。

| | Canvas | WebGL |
|---|--------|-------|
| **性能** | 较低 | 高 |
| **兼容性** | 极好 | 好（极少数旧设备不支持） |
| **Shader 支持** | 不支持 | 支持 |
| **适合场景** | 简单游戏、需要极致兼容 | 大多数游戏 |

对于大多数 Phaser 游戏开发，WebGL 是更好的选择，这也是 `Phaser.AUTO` 优先选 WebGL 的原因。

但如果你的游戏对象很少、画面极简，Canvas 也完全足够。

## Self-Check

**Q1.** Renderer 在游戏引擎里的职责是什么？

> **答：** Renderer（渲染器）负责每帧把所有游戏对象的状态（位置、图像等）绘制到屏幕上，让玩家看到游戏画面。

---

**Q2.** Canvas Renderer 和 WebGL Renderer 最核心的区别是什么？

> **答：** Canvas Renderer 由 CPU 执行绘图，兼容性好但性能较低；WebGL Renderer 利用 GPU 加速，性能强，还支持 Shader，但极少数旧设备不支持。

---

**Q3.** 什么是 GPU？它为什么比 CPU 更适合处理游戏渲染？

> **答：** GPU（图形处理器）是专门为图形计算设计的芯片，拥有大量并行核心，可以同时处理大量像素计算。游戏渲染需要每帧处理成千上万个像素，GPU 的并行能力比 CPU 更适合这种工作。

---

**Q4.** `Phaser.AUTO` 会如何决定使用哪种渲染器？

> **答：** Phaser.AUTO 先检测浏览器是否支持 WebGL：支持则使用 WebGL（性能更好）；不支持则 fallback（回退）到 Canvas，保证兼容性。

---

**Q5.** 什么是 Shader？哪种渲染器支持它？

> **答：** Shader（着色器）是运行在 GPU 上的程序，可以实现光照、模糊、色调变换等特殊视觉效果。只有 WebGL Renderer 支持 Shader，Canvas Renderer 不支持。
