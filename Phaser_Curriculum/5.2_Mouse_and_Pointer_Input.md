# 5.2 — Mouse and Pointer Input

## 大局观 (High-Level Map)

在 5.1 你学了怎么处理键盘。键盘的特点是"按键 = 离散的开/关状态"。

鼠标不一样 — 它有**位置（坐标）**，还有**点击事件**，而且现代游戏还需要支持触控屏。

Phaser 的解决方案是：用一个叫 **Pointer（指针）** 的统一抽象，把鼠标和触控都归为同一套 API，你写一套代码，两种设备都能用。

```
大图：
[ 鼠标（Mouse）]  ─┐
                   ├→ [ Pointer（抽象层）] → 你的代码
[ 触控（Touch）] ─┘
```

## 1. 为什么需要 Pointer 抽象？

如果没有统一抽象，你需要分别写：

```javascript
// 桌面版：监听鼠标
canvas.addEventListener('mousedown', (e) => { ... });
canvas.addEventListener('mousemove', (e) => { ... });

// 移动版：监听触控
canvas.addEventListener('touchstart', (e) => { ... });
canvas.addEventListener('touchmove', (e) => { ... });
```

同样的功能写两遍，维护两套逻辑，还要处理各种边缘情况。

Phaser 的 **Pointer** 把两者统一：鼠标点击 = 触控点击，鼠标移动 = 触控移动。你只写一套代码，自动兼容两种输入方式。

---

## 2. Pointer 是什么？

**Pointer** 是 Phaser 对"屏幕上一个接触点"的抽象表示。

- 在桌面电脑上：Pointer = 鼠标光标
- 在触控屏上：Pointer = 手指

Phaser 默认提供一个主 Pointer，通过 `this.input.activePointer` 或 `this.input.pointer1` 访问。

```javascript
// 获取当前活跃的 Pointer
const pointer = this.input.activePointer;
```

---

## 3. Pointer 的位置

Pointer 随时追踪它在屏幕上的当前坐标，坐标系和 Sprite 一样（左上角为原点）。

```javascript
update() {
    const pointer = this.input.activePointer;

    console.log(pointer.x);  // Pointer 的 X 坐标（像素）
    console.log(pointer.y);  // Pointer 的 Y 坐标（像素）
}
```

这个坐标是**屏幕坐标（screen space）**，指的是在 Canvas 上的位置。

**实际用途举例：**
- 让一个准星图片跟随鼠标移动
- 计算玩家点击的方向，让子弹朝那个方向飞

```javascript
update() {
    const pointer = this.input.activePointer;
    crosshair.setPosition(pointer.x, pointer.y);
}
```

## 4. 检测点击（Click Detection）

### isDown — 正在按住

```javascript
update() {
    if (this.input.activePointer.isDown) {
        // 鼠标左键正在被按住，或手指正在触碰屏幕
    }
}
```

和键盘的 `isDown` 一样：按住期间每帧都是 `true`。

### 监听点击事件

除了在 `update()` 里 polling，Phaser 也支持用**事件监听**来捕捉点击：

```javascript
// 在 create() 里注册
this.input.on('pointerdown', (pointer) => {
    // 玩家在 (pointer.x, pointer.y) 按下了鼠标/触控
    console.log('点击位置：', pointer.x, pointer.y);
});

this.input.on('pointerup', (pointer) => {
    // 玩家松开了鼠标/手指
});

this.input.on('pointermove', (pointer) => {
    // Pointer 正在移动
});
```

**`pointerdown` vs `isDown` 的区别：**

| | `isDown`（polling） | `pointerdown`（事件） |
|---|---|---|
| 触发方式 | 在 `update()` 里主动查询 | 按下时自动触发一次 |
| 适合场景 | 持续按住时持续执行 | 一次性点击动作 |

两种方式可以根据需要混用。

## 5. Pointer 的其他状态

| 属性 / 事件 | 说明 |
|------------|------|
| `pointer.isDown` | 当前是否按住 |
| `pointer.x`, `pointer.y` | 当前位置 |
| `pointer.leftButtonDown()` | 鼠标左键是否按住（鼠标专用） |
| `pointer.rightButtonDown()` | 鼠标右键是否按住（鼠标专用） |
| `pointer.distance` | 从按下到现在移动的总距离（适合检测滑动） |
| `'pointerdown'` 事件 | 按下瞬间触发 |
| `'pointerup'` 事件 | 松开瞬间触发 |
| `'pointermove'` 事件 | 移动时持续触发 |

**注意：** `leftButtonDown()` 和 `rightButtonDown()` 是鼠标特有的，触控屏没有"左键/右键"之分，不要在跨平台代码里用这两个。

## Self-Check

**Q1.** Phaser 为什么要用 Pointer 这个抽象层，而不是直接让开发者分别处理鼠标和触控事件？

> **答：** 鼠标和触控虽然是不同的硬件，但"在屏幕某个位置产生点击/移动"这个行为是相同的。Pointer 把两者统一，让开发者只写一套代码就能同时支持桌面鼠标和移动触控，避免重复维护两套逻辑。

---

**Q2.** `pointer.isDown` 和监听 `'pointerdown'` 事件有什么区别？分别适合什么场景？

> **答：**
> - `pointer.isDown` 是 polling，在 `update()` 里每帧查询，适合"按住期间每帧都要执行"的持续动作（例如按住鼠标持续射击）。
> - `'pointerdown'` 事件是在按下的瞬间自动触发一次，适合一次性动作（例如点击按钮、放置物体）。

---

**Q3.** 下面的代码想让一个图标跟随鼠标移动，它写在 `update()` 里。有没有问题？

```javascript
update() {
    icon.setPosition(this.input.activePointer.x, this.input.activePointer.y);
}
```

> **答：** 没有问题。`pointer.x` 和 `pointer.y` 每帧都反映 Pointer 的最新位置，在 `update()` 里读取并设置 icon 的位置，可以让图标实时跟随鼠标/触控点移动。这是一个标准的用法。
