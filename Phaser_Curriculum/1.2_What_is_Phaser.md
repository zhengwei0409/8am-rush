# 1.2 — What is Phaser

## 大局观 (High-Level Map)

上一节我们学了什么是 Game Engine（游戏引擎）。现在我们进入正题：**Phaser** 是什么？

Phaser 是一个具体的游戏引擎。但它不是像 Unity 或 Unreal 那种"下载安装、打开软件"的引擎——它是一个运行在**浏览器**里的 JavaScript 游戏框架。

理解 Phaser 的关键，是先搞清楚它生活在哪个世界里。

---

## 本节结构

1. Web 生态中的 Phaser 在哪里？

## 1. Web 生态中的 Phaser 在哪里？

你可能已经知道：网页是由 HTML、CSS、JavaScript 组成的。

- **HTML** — 定义网页的结构（有哪些元素）
- **CSS** — 定义网页的样式（元素长什么样）
- **JavaScript** — 定义网页的行为（元素怎么动）

Phaser 就是一个用 JavaScript 写的游戏框架，游戏直接运行在浏览器里，不需要安装任何客户端程序。

---

## 2. Phaser 的定义

> **Phaser** 是一个基于 HTML5 的 2D 游戏框架（framework），用 JavaScript 或 TypeScript 编写，游戏在浏览器中运行。

先解释两个词：

**HTML5** — HTML 的第5个版本，引入了 `<canvas>` 标签，这让浏览器第一次拥有了"画任意图形"的能力，从而使浏览器游戏成为可能。

**Framework（框架）** — 框架和 Library（库）类似，都是别人写好的代码让你复用。区别是：框架会规定你的代码结构（你按框架的规则写代码），而库则是你主动去调用它。Phaser 介于两者之间，通常被称为框架。

---

## 3. 五个核心概念逐一拆解

### 3.1 HTML5 Game Framework

Phaser 依赖 HTML5 提供的两个关键技术：
- `<canvas>` — 一块可以用代码随意绘制图形的区域
- **Web APIs** — 浏览器提供的接口，比如音频、输入、计时器

Phaser 帮你封装了直接操作这些 API 的复杂性，你不需要亲自写 `canvas.getContext('2d').drawImage(...)` 这样的底层代码。

---

### 3.2 JavaScript / TypeScript Based

Phaser 的游戏逻辑用 **JavaScript** 或 **TypeScript** 编写。

- **JavaScript** — 浏览器原生支持的脚本语言
- **TypeScript** — JavaScript 的超集（superset），加入了类型系统（type system），让代码更严谨、更容易维护

**Superset** 的意思是：TypeScript 包含所有 JavaScript 的语法，同时额外增加了新功能。所有合法的 JavaScript 代码，也是合法的 TypeScript 代码。

Phaser 3 两者都支持，Phaser 4 更偏向 TypeScript。

---

### 3.3 Browser-Based Runtime

**Runtime（运行时）** 指的是代码实际运行的环境。

Phaser 的 Runtime 就是浏览器本身（Chrome、Firefox、Safari 等）。这意味着：

- 游戏不需要安装
- 玩家打开一个网址就能玩
- 跨平台：同一份代码在 Windows、macOS、Android、iOS 都能运行，只要有浏览器

这和 Unity 不同——Unity 游戏通常需要编译成特定平台的可执行文件。

---

### 3.4 Canvas and WebGL Rendering

Phaser 支持两种渲染方式，都是由浏览器提供的：

| 渲染方式 | 说明 |
|----------|------|
| **Canvas** | 用 CPU 绘制 2D 图形，兼容性好，性能相对低 |
| **WebGL** | 使用 GPU 加速，性能强，支持更复杂的视觉效果 |

**GPU（Graphics Processing Unit，图形处理器）** — 专门用来处理图形计算的芯片，比 CPU 更适合并行处理大量像素。

**WebGL（Web Graphics Library）** — 浏览器提供的接口，允许 JavaScript 直接调用 GPU 的能力。

Phaser 默认会自动选择：如果浏览器支持 WebGL，就用 WebGL；否则 fallback 到 Canvas。这个模式叫 **AUTO**。

---

### 3.5 Open-Source Library

Phaser 是 **Open-Source（开源）** 的，意思是：

- 代码是公开的，任何人都可以查看
- 免费使用（对个人和商业项目）
- 由社区贡献和维护

Phaser 托管在 GitHub 上，你可以直接查看它的源码、提交 bug 报告，或者参与贡献。

---

## Self-Check

**Q1.** Phaser 运行在哪里？它和 Unity 这类引擎最大的区别是什么？

> **答：** Phaser 运行在浏览器里（browser-based runtime）。Unity 游戏需要编译成平台可执行文件安装运行，而 Phaser 游戏只需要打开网址，浏览器直接运行，无需安装。

---

**Q2.** HTML5 中，哪项技术让浏览器游戏变成可能？

> **答：** HTML5 引入的 `<canvas>` 标签，让浏览器可以用代码随意绘制图形，这是浏览器游戏的基础。

---

**Q3.** TypeScript 是什么？它和 JavaScript 是什么关系？

> **答：** TypeScript 是 JavaScript 的 superset（超集），在 JavaScript 基础上增加了类型系统。所有合法的 JavaScript 代码都是合法的 TypeScript 代码，但 TypeScript 提供了更严格的类型检查，使代码更易维护。

---

**Q4.** Phaser 支持哪两种渲染方式？它们的主要区别是什么？

> **答：** Canvas（用 CPU 绘制，兼容性好，性能较低）和 WebGL（用 GPU 加速，性能更强，支持复杂视觉效果）。Phaser 默认使用 AUTO 模式自动选择。

---

**Q5.** 什么是 Open-Source？Phaser 是开源的代表着什么？

> **答：** Open-Source（开源）指代码公开、免费，任何人可以查看和使用。Phaser 开源意味着可以免费用于个人和商业项目，并且你可以查看它的源码了解内部实现。
