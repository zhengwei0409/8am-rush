# 2.1 — Phaser Project Structure

## 大局观 (High-Level Map)

你已经知道 Phaser 是什么、游戏循环是怎么运作的。现在要进入更实际的问题：**一个 Phaser 项目的文件是如何组织的？**

在你写第一行游戏代码之前，你需要先有一套文件结构。就像盖房子之前要有地基和框架，游戏项目也有它的最小必要组成部分。

这一节只讲**概念上的文件布局**，不需要你动手配置任何工具。

## 1. 一个 Phaser 项目最少需要什么？

Phaser 是一个运行在浏览器里的框架。浏览器加载的起点永远是一个 HTML 文件。所以一个最简单的 Phaser 项目，至少需要：

1. 一个 HTML 入口文件
2. Phaser 的库文件（或引用）
3. 你自己写的游戏逻辑 JS 文件
4. 游戏用到的图片、音效等资源
5. 一个本地服务器来运行它

---

## 2. 五个核心组成部分逐一拆解

### 2.1 index.html — 入口文件（Entry Point）

**Entry Point（入口）** 是整个程序开始执行的地方。

对于浏览器来说，入口就是 HTML 文件。浏览器打开你的项目，第一件事就是读取 `index.html`。

这个 HTML 文件的职责很简单：
- 告诉浏览器"加载 Phaser 库"
- 告诉浏览器"加载我的游戏主文件"
- 提供一个 `<canvas>` 容器让 Phaser 把游戏画在里面

```html
<!-- 这是 index.html 的简化结构 -->
<!DOCTYPE html>
<html>
  <body>
    <script src="phaser.js"></script>   <!-- 加载 Phaser 库 -->
    <script src="main.js"></script>     <!-- 加载你的游戏代码 -->
  </body>
</html>
```

---

### 2.2 Phaser Library Import — 引入 Phaser 库

Phaser 本身是一个 JavaScript 文件（或通过网络引用的 URL）。在你的游戏代码能使用 Phaser 提供的功能之前，必须先把这个库加载进来。

有两种常见方式：

| 方式 | 说明 |
|------|------|
| 本地文件 | 下载 `phaser.js` 放在项目文件夹里，用 `<script src="phaser.js">` 引入 |
| CDN | 直接引用网络上的 Phaser 文件，不需要下载（需要联网） |

**CDN（Content Delivery Network，内容分发网络）** 是一种把文件放在全球服务器上、让用户就近下载的服务。

重要：Phaser 库必须在你的游戏代码**之前**加载，否则你的代码找不到 `Phaser` 这个变量。

---

### 2.3 Main JS File — 主游戏逻辑文件

这是你**真正写游戏代码**的地方。通常叫 `main.js` 或 `game.js`。

这个文件负责：
- 创建 Phaser 的 Game 对象（配置游戏的基本参数）
- 注册游戏用到的所有 Scene（场景）
- 启动游戏

这个文件是 Phaser 项目的"大脑"，所有游戏逻辑的起点都在这里（Scene 的具体逻辑通常会拆分到其他文件）。

---

### 2.4 Assets Folder — 资源文件夹

**Assets（资源）** 是游戏用到的所有外部文件，比如：
- 图片（角色、背景、UI）
- 音效和音乐
- 字体文件
- 地图数据

这些文件通常放在一个叫 `assets/` 的文件夹里，按类型再细分：

```
assets/
  images/
    player.png
    enemy.png
    background.png
  audio/
    jump.mp3
    bgm.ogg
```

Phaser 在游戏运行时会通过网络请求来加载这些文件。把它们统一放在 `assets/` 文件夹里，方便管理和引用路径。

---

### 2.5 Local Dev Server — 本地开发服务器

这是一个容易被忽视但非常重要的细节。

你不能直接双击 `index.html` 在浏览器里打开来运行 Phaser 游戏。原因是：

浏览器有一个安全限制叫 **Same-Origin Policy（同源策略）**。当你用 `file://` 协议（直接打开本地文件）时，浏览器会拒绝 JavaScript 加载其他本地文件（比如图片、音效）。

**Local Dev Server（本地开发服务器）** 解决这个问题：它在你的电脑上模拟一个真实的 Web 服务器，让浏览器通过 `http://localhost` 访问你的文件，绕过安全限制。

常用的本地开发服务器工具：
- VS Code 插件 **Live Server**（最简单，点击启动）
- `npx serve`（命令行工具）
- `python -m http.server`（用 Python 启动）

> 总结：你的开发流程是 → 启动本地服务器 → 浏览器访问 `http://localhost:端口号` → 看到游戏运行

---

## 3. 文件结构总览

一个最简单的 Phaser 项目长这样：

```
my-game/
├── index.html        ← 浏览器入口，加载 Phaser 和游戏代码
├── phaser.js         ← Phaser 库文件
├── main.js           ← 你的游戏主逻辑
└── assets/
    ├── images/
    │   └── player.png
    └── audio/
        └── jump.mp3
```

浏览器的加载顺序：
```
浏览器打开 index.html
    → 加载 phaser.js（Phaser 库就绪）
    → 加载 main.js（游戏启动）
    → 游戏在 preload() 里请求 assets/ 里的文件
    → 游戏开始运行
```

## Self-Check

**Q1.** 为什么浏览器项目一定要有 `index.html`？它的作用是什么？

> **答：** 浏览器加载网页的入口永远是 HTML 文件。`index.html` 负责告诉浏览器要加载哪些 JavaScript 文件（Phaser 库和游戏代码），是整个项目的起点。

---

**Q2.** 为什么 Phaser 库文件必须在游戏代码之前加载？

> **答：** 游戏代码依赖 Phaser 提供的功能（比如 `Phaser.Game`）。如果 Phaser 还没加载，游戏代码运行时找不到 `Phaser` 变量，会报错。

---

**Q3.** 什么是 Assets？游戏项目里通常把它们放在哪里？

> **答：** Assets 是游戏用到的外部资源文件，比如图片和音效。通常统一放在 `assets/` 文件夹里，按类型（images/、audio/）细分管理。

---

**Q4.** 为什么不能直接双击 `index.html` 来运行 Phaser 游戏？需要用什么代替？

> **答：** 浏览器用 `file://` 协议打开本地文件时，Same-Origin Policy 会阻止 JavaScript 加载其他本地文件（图片、音效等）。需要启动一个 Local Dev Server（如 VS Code 的 Live Server），通过 `http://localhost` 访问项目才能正常运行。

---

**Q5.** 以下文件各自的职责是什么？`index.html`、`main.js`、`assets/`

> **答：**
> - `index.html`：浏览器入口，负责加载 Phaser 库和游戏代码
> - `main.js`：游戏主逻辑，创建 Game 对象、注册 Scene、启动游戏
> - `assets/`：存放游戏用到的外部资源（图片、音效等）
