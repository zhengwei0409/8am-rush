## 1. Phaser 怎么处理键盘输入？

浏览器原生支持键盘事件（`keydown`、`keyup`），但直接用这些事件做游戏输入有问题：

- 原生事件是**异步触发**的，和游戏的 `update()` 循环不同步
- 你需要自己管理"哪些键现在被按住"的状态
- 处理多个键同时按下很麻烦

Phaser 的 **Keyboard Plugin** 把这些细节封装好了。它在后台监听浏览器键盘事件，并维护一张"每个键当前状态"的表。

你在 `update()` 里**主动询问（polling）**某个键的状态，而不是被动等待事件触发。

```
浏览器原生事件 → Phaser Keyboard Plugin → 你在 update() 里 polling
                  （后台维护键状态）          （主动查询，同步）
```

## 2. Keyboard Plugin

**Keyboard Plugin** 是 Phaser Scene 里内置的键盘管理模块，通过 `this.input.keyboard` 访问。

不需要手动引入或初始化，Phaser 游戏启动后它就自动可用。

---

## 3. Key Object 是什么？

**Key Object** 是代表一个具体键盘按键的对象。你用它来追踪那个键的当前状态。

创建 Key Object：
```javascript
// 在 create() 里
this.spaceKey = this.input.keyboard.addKey(Phaser.Input.Keyboard.KeyCodes.SPACE);
this.wKey = this.input.keyboard.addKey(Phaser.Input.Keyboard.KeyCodes.W);
```

`KeyCodes` 是 Phaser 内置的键名枚举，包含所有常见按键：
```
KeyCodes.SPACE    KeyCodes.ENTER    KeyCodes.ESC
KeyCodes.UP       KeyCodes.DOWN     KeyCodes.LEFT    KeyCodes.RIGHT
KeyCodes.W        KeyCodes.A        KeyCodes.S       KeyCodes.D
KeyCodes.SHIFT    KeyCodes.CTRL     KeyCodes.ALT
```

一个 Key Object 只代表一个键。你需要多少个键，就创建多少个 Key Object。

## 4. isDown — 检测"正在按住"

**`isDown`** 是 Key Object 的一个属性，值是 `true` 或 `false`：

- `true` — 这个键**目前正在被按住**
- `false` — 这个键目前没有被按

在 `update()` 里使用：
```javascript
update() {
    if (this.spaceKey.isDown) {
        // 玩家正在按住空格键
        player.setVelocityY(-300);  // 持续向上飞
    }
}
```

因为 `update()` 每帧都运行，只要玩家按住不放，`isDown` 每帧都是 `true`，对应的代码每帧都会执行。

**适合的场景：** 持续性动作，例如"按住方向键持续移动"、"按住射击键持续开枪"。

---

## 5. JustDown — 检测"刚刚按下"

**`JustDown`** 检测一个键是否**在这一帧刚刚被按下**（从"没按"变成"按下"的那一瞬间）。

```javascript
update() {
    if (Phaser.Input.Keyboard.JustDown(this.spaceKey)) {
        // 空格键在这一帧刚刚被按下
        player.jump();  // 只触发一次
    }
}
```

**`isDown` vs `JustDown` 的关键区别：**

| | `isDown` | `JustDown` |
|---|---|---|
| 触发条件 | 键正在被按住 | 键刚刚从"无"变成"按下" |
| 持续按住时 | 每帧都是 `true` | 只有第一帧是 `true` |
| 适合场景 | 持续移动、持续射击 | 跳跃、切换武器、暂停 |

**为什么跳跃要用 JustDown 而不是 isDown？**

如果用 `isDown`，玩家按住空格不放，角色会在空中每帧都触发跳跃，飞到天上去。
用 `JustDown`，无论按住多久，跳跃只触发一次。

## 6. Cursor Keys 快捷方式

Phaser 提供了一个便捷方法，一次性创建方向键（上下左右）+ Shift + Alt 的 Key Object：

```javascript
// 在 create() 里
this.cursors = this.input.keyboard.createCursorKeys();
```

然后在 `update()` 里直接用：
```javascript
update() {
    if (this.cursors.left.isDown) {
        player.setVelocityX(-200);
    } else if (this.cursors.right.isDown) {
        player.setVelocityX(200);
    }

    if (Phaser.Input.Keyboard.JustDown(this.cursors.up)) {
        player.jump();
    }
}
```

`createCursorKeys()` 返回一个包含这些属性的对象：
```
cursors.up
cursors.down
cursors.left
cursors.right
cursors.shift
cursors.space
```

这是做平台跳跃游戏（platformer）最快的起点。

## Self-Check

**Q1.** 什么是 Polling（轮询）？Phaser 的键盘输入为什么用 polling 而不是直接监听浏览器键盘事件？

> **答：** Polling 是指在固定时间点**主动查询**某个状态，而不是被动等待事件通知。Phaser 用 polling 是因为游戏的 `update()` 每帧运行，你需要每帧都知道键盘状态来做出正确响应。直接监听浏览器键盘事件是异步的，和游戏循环不同步，难以处理"按住不放"的持续输入状态。

---

**Q2.** 玩家按住 W 键 3 秒不放。在这 3 秒里，`isDown` 和 `JustDown` 各自的值是什么？

> **答：**
> - `isDown`：在整个 3 秒内（每一帧）都是 `true`
> - `JustDown`：**只有按下的第一帧**是 `true`，之后每帧都是 `false`

---

**Q3.** 为什么"跳跃"应该用 `JustDown` 而不是 `isDown`？

> **答：** 如果用 `isDown`，玩家按住跳跃键不放，`update()` 每帧都会检测到 `true`，角色会在每一帧都触发跳跃，产生不断向上飞的 bug。用 `JustDown`，无论玩家按住多久，跳跃动作只在按下的第一帧触发一次，符合玩家的预期行为。
