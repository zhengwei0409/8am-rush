# 3.1 — What is a Scene

## 大局观 (High-Level Map)

你已经知道了 Phaser 项目的文件结构，也知道了如何用 Config 启动游戏。Config 里有一个字段叫 `scene`，我们当时只是简单提了一下。

现在我们要深入理解：**Scene（场景）是什么？**

Scene 是 Phaser 里最核心的概念之一。整个游戏的所有内容——主菜单、游戏关卡、结算画面——都是通过 Scene 来组织的。

---

## 本节结构

1. Scene 的定义
2. Isolated State — 场景有自己的独立状态
3. Independent Lifecycle — 场景有自己的独立生命周期
4. Scene as Screen — 把场景理解为"一个画面"
5. Multiple Scenes per Game — 一个游戏可以有多个场景
6. Self-Check

---

## 1. Scene 的定义

> **Scene（场景）** 是 Phaser 游戏中一个自包含（self-contained）的游戏状态单元。

**Self-contained（自包含）** 的意思是：一个 Scene 拥有自己运行所需的一切——自己的资源、自己的游戏对象、自己的逻辑——不依赖其他 Scene 才能工作。

你可以把 Scene 想象成一个独立的"房间"。每个房间有自己的家具（游戏对象）和规则（游戏逻辑），你进入不同的房间就体验不同的内容。

---

## 2. Isolated State — 隔离的状态

**State（状态）** 是指某个时刻，游戏中所有变量的当前值，比如：玩家的血量、分数、敌人的位置。

**Isolated（隔离）** 的意思是：每个 Scene 维护自己的状态，和其他 Scene 互不干扰。

举个例子：
- `GameScene` 里有玩家的位置、血量、分数
- `MenuScene` 里有按钮的状态、光标位置

这两个 Scene 的状态完全分开。`MenuScene` 不需要知道玩家有多少血量，`GameScene` 也不需要知道菜单里有什么按钮。

这种隔离让每个 Scene 更简单、更容易维护。

---

## 3. Independent Lifecycle — 独立的生命周期

**Lifecycle（生命周期）** 在 1.1 节提过：指一个事物从创建到销毁经历的各个阶段。

每个 Scene 都有自己的生命周期，完全独立运行：

```
Scene 被启动
    ↓
preload()  — 加载这个 Scene 需要的资源
    ↓
create()   — 初始化游戏对象和逻辑
    ↓
update()   — 每帧持续执行，处理游戏逻辑
    ↓
Scene 被停止或销毁
```

关键点：Scene A 的生命周期和 Scene B 的生命周期**互不影响**。Scene A 在运行，Scene B 可以同时处于停止状态。

这三个方法（`preload`、`create`、`update`）的细节会在 3.2 节详细讲。

---

## 4. Scene as Screen — 场景即"画面"

理解 Scene 最直观的方式：**把每个 Scene 当作游戏的一个独立"画面"或"屏幕"**。

一个典型的游戏有这些"画面"：

| 画面 | 对应 Scene |
|------|-----------|
| 游戏加载中... | Boot Scene |
| 主菜单（开始/设置/退出） | Menu Scene |
| 游戏进行中 | Game Scene |
| 游戏暂停 | Pause Scene |
| 游戏结算（你的分数是...） | GameOver Scene |

玩家每次看到的是**当前激活的 Scene**，切换画面就是切换 Scene。

---

## 5. Multiple Scenes per Game — 一个游戏有多个场景

一个 Phaser 游戏通常包含**多个 Scene**，在 Config 的 `scene` 数组里全部注册：

```javascript
const config = {
    scene: [BootScene, MenuScene, GameScene, GameOverScene]
};
```

这些 Scene 不是同时运行的（默认情况下），而是按需激活：
- 游戏启动 → `BootScene` 运行
- 资源加载完毕 → 切换到 `MenuScene`
- 玩家点击开始 → 切换到 `GameScene`
- 玩家死亡 → 切换到 `GameOverScene`

**Scene Manager（场景管理器）** 是 Phaser 内置的系统，负责追踪所有已注册的 Scene、控制它们的启动和停止。你不需要自己实现这个系统，Phaser 帮你做好了。

---

## Self-Check

**Q1.** 什么是 Scene？"Self-contained" 是什么意思？

> **答：** Scene 是 Phaser 中一个自包含的游戏状态单元。Self-contained 意味着每个 Scene 拥有自己运行所需的一切（资源、对象、逻辑），不依赖其他 Scene 才能工作。

---

**Q2.** "Isolated State" 中的 Isolated 指的是什么？为什么这种隔离是好事？

> **答：** Isolated 指每个 Scene 维护自己的独立状态，和其他 Scene 互不干扰。好处是每个 Scene 只需关注自己的逻辑，更简单、更容易维护，不会因为其他 Scene 的状态改变而出错。

---

**Q3.** 用"画面"来理解 Scene：一个典型游戏的主菜单、游戏进行中、游戏结算，各对应什么？

> **答：** 主菜单对应 Menu Scene，游戏进行中对应 Game Scene，游戏结算对应 GameOver Scene。每个独立画面对应一个独立的 Scene。

---

**Q4.** Scene Manager 的作用是什么？

> **答：** Scene Manager 是 Phaser 内置的场景管理系统，负责追踪所有已注册的 Scene，控制它们的启动、停止和切换。开发者不需要自己实现，Phaser 提供好了。

---

**Q5.** 一个游戏在 Config 里注册了 `[BootScene, MenuScene, GameScene]`，这三个 Scene 是同时运行的吗？

> **答：** 不是。默认情况下，这些 Scene 按需激活，同一时间通常只有一个 Scene 处于运行状态。游戏启动时 BootScene 最先运行，之后按逻辑切换到其他 Scene。
