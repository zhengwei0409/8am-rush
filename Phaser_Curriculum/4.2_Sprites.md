# 4.2 — Sprites

## 大局观 (High-Level Map)

在 4.1 你学会了如何把图片**加载进内存**。现在的问题是：加载进来之后，怎么把它**显示在屏幕上**？

答案是：用 **Sprite**。

Sprite 是 Phaser 里最基本的"视觉游戏对象"。它把一张图片放到屏幕上的某个位置，并且可以被移动、缩放、旋转。理解 Sprite 是后面 Animation（4.3）和 Physics（Section 6）的基础。

```
大图：
[ Asset 加载完毕 ] → [ 创建 Sprite，绑定图片 ] → [ Sprite 出现在屏幕上 ]
                                                        ↓
                                              [ 可以移动 / 缩放 / 旋转 ]
```

## 1. 什么是 Sprite？

**Sprite** 是一个能在屏幕上显示图片的游戏对象。

在游戏里，几乎所有你能看到的东西都是 Sprite：玩家角色、敌人、子弹、地板砖块、道具。

Sprite 有两个核心职责：
1. **"我长什么样"** — 绑定一张图片（Texture）
2. **"我在哪里"** — 有 x, y 坐标，确定它在屏幕上的位置

---

## 2. Texture（纹理）是什么？

**Texture** 就是 Sprite 显示的那张图片数据。

当你在 `preload()` 里加载了一张图片：
```javascript
this.load.image('player', 'assets/player.png');
```

Phaser 会把这张图片的像素数据存进**Texture Manager（纹理管理器）**，并用你给的 key `'player'` 来标记它。

创建 Sprite 时，你用 key 告诉 Sprite 该用哪张 Texture：
```javascript
this.add.sprite(x, y, 'player');
```

**Texture 和 Sprite 的关系：**
- Texture 是图片数据本身（存在内存里）
- Sprite 是一个"容器"，它**引用**一个 Texture 来决定自己画什么

同一张 Texture 可以被多个 Sprite 引用，内存里只存一份图片数据，节省资源。

---

## 3. Sprite 的位置（x, y）

Sprite 的位置用 **x** 和 **y** 两个数字表示，单位是**像素（pixel）**。

Phaser 坐标系（你在 1.4 学过）：
- 原点 `(0, 0)` 在**左上角**
- x 向右增大
- y 向下增大

```
(0,0) ──────→ x
  │
  │
  ↓
  y
```

创建 Sprite：
```javascript
this.add.sprite(100, 200, 'player');
//              ↑x   ↑y
```

这会把 Sprite 放在距离左边 100px、距离顶部 200px 的位置。

### Origin（原点/锚点）

Sprite 的 x, y 默认指的是 Sprite **中心点**的位置，不是左上角。

这个中心点叫做 **Origin**，默认值是 `(0.5, 0.5)`，代表图片宽度的 50% 和高度的 50%（即中心）。

你可以改变 Origin，但默认中心对齐在大多数情况下是最直观的。

---

## 4. Sprite 的其他变换属性

Sprite 除了位置，还有以下可以改变的属性：

| 属性 | 说明 | 默认值 |
|------|------|--------|
| `x`, `y` | 位置（像素） | 创建时指定 |
| `scaleX`, `scaleY` | 缩放比例（1 = 原始大小） | 1 |
| `rotation` | 旋转角度（弧度 radians） | 0 |
| `alpha` | 透明度（0 = 完全透明，1 = 完全不透明） | 1 |
| `visible` | 是否可见（true/false） | true |
| `flipX` | 水平翻转 | false |

这些属性让 Sprite 成为一个**可变换的实体（Transformable Entity）**，不只是静态贴图。

---

## 5. Sprite 是一个 Game Object

在 Phaser 里，**Game Object** 是所有可以放在 Scene 里的东西的统称。

Sprite 是 Game Object 的一种，其他还有：
- Text（文字，4.6 会学）
- Image（静态图片，比 Sprite 更轻量）
- Rectangle、Circle（几何形状）

Sprite 和 Image 的区别：
- **Image** — 只显示静态图片，没有 Animation 支持，性能开销更小
- **Sprite** — 支持 Animation（帧动画），功能更多

如果你的图片不需要动画，用 `this.add.image()` 比 `this.add.sprite()` 更高效。

---

## Self-Check

**Q1.** Texture 和 Sprite 有什么区别？一张 Texture 可以被多少个 Sprite 使用？

> **答：** Texture 是图片的像素数据，存储在内存里。Sprite 是一个游戏对象，它**引用**一个 Texture 来决定自己显示什么。一张 Texture 可以被**无数个** Sprite 同时引用，内存里只保留一份数据，不会重复占用空间。

---

**Q2.** 下面代码创建的 Sprite，其中心点在屏幕的哪个位置？

```javascript
this.add.sprite(400, 300, 'enemy');
```

> **答：** Sprite 的中心点在距左边 400px、距顶部 300px 的位置。因为 Origin 默认是 (0.5, 0.5)，所以 x=400, y=300 指的是 Sprite 图片的中心，而不是左上角。

---

**Q3.** 什么时候应该用 `this.add.image()` 而不是 `this.add.sprite()`？

> **答：** 当图片是静态的、不需要播放帧动画时，应该用 `this.add.image()`。它比 Sprite 更轻量，性能开销更小。Sprite 的额外功能（Animation）在不需要动画时是多余的负担。
