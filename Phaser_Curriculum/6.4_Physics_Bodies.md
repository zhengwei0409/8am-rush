# 6.4 — Physics Bodies

## 大局观 (High-Level Map)

在 6.2 你学了 Arcade Physics 系统。在 6.3 你学了 Matter.js。

但两者有一个共同的核心概念还没讲清楚：**Physics Body（物理体）**。

Sprite 是"视觉的" — 它负责显示图片。Physics Body 是"物理的" — 它负责运动和碰撞。两者是分开的，但绑定在一起。

理解 Body 是理解"物理引擎到底在操作什么"的关键。

```
大图：
┌─────────────────────────────────┐
│           Sprite（游戏对象）     │
│                                 │
│  [图片显示层]   [Physics Body]  │
│  (你眼睛看到的)  (物理引擎操作的) │
└─────────────────────────────────┘
```

## 1. 什么是 Physics Body？

**Physics Body** 是物理引擎为一个游戏对象维护的"物理数据容器"。

它存储这个对象的：
- 当前速度（velocity）
- 当前加速度（acceleration）
- 重力影响
- 碰撞边界（大小和形状）
- 是否被阻挡（例如 `body.blocked.down = true` 表示脚下有地面）

物理引擎每帧更新所有 Body 的数据，然后把结果反映到对应 Sprite 的位置上。

```
物理引擎更新 Body.velocity, Body.position
           ↓
Sprite.x = Body.position.x
Sprite.y = Body.position.y
```

你改变 Sprite 位置，是通过改变 Body 的速度，而不是直接改 `sprite.x`。

---

## 2. Body 和 Sprite 的关系

一个带物理的 Sprite 有两层：

```
Sprite
  ├── 视觉层（你在屏幕上看到的图片）
  └── Physics Body（物理引擎追踪的数据）
```

它们**独立存在，但绑定同步**：
- 物理引擎只认识 Body，不认识图片
- 每帧结束后，Phaser 把 Body 的位置同步给 Sprite 的显示位置
- 你永远不会直接移动图片，你是移动 Body，图片跟着走

通过 `sprite.body` 访问这个 Sprite 的 Physics Body：
```javascript
const player = this.physics.add.sprite(100, 100, 'player');
console.log(player.body);  // ArcadeBody 对象
```

## 3. Static Body vs Dynamic Body

这是 Physics Body 最重要的分类。

### Dynamic Body（动态体）

**Dynamic Body** 会被物理引擎完全模拟：
- 受重力影响，会下落
- 速度会让它移动
- 会被其他物体推开
- 碰到东西会停下来

```javascript
// 创建带 Dynamic Body 的 Sprite
const player = this.physics.add.sprite(100, 100, 'player');
// 默认就是 Dynamic Body
```

适合：玩家角色、敌人、子弹、掉落的道具。

### Static Body（静态体）

**Static Body** 是永远不动的物理对象：
- 不受重力影响
- 速度对它无效
- 其他物体碰到它会被阻挡，但它自己不移动
- 性能开销极低（物理引擎不需要每帧更新它的位置）

```javascript
// 创建带 Static Body 的 Sprite
const platform = this.physics.add.staticSprite(400, 500, 'platform');
// 或者
const platform = this.physics.add.sprite(400, 500, 'platform');
platform.body.setImmovable(true);
platform.body.setAllowGravity(false);
```

适合：地板、墙壁、平台、障碍物。

**直觉对比：**

| | Dynamic Body | Static Body |
|---|---|---|
| 受重力 | ✓ | ✗ |
| 速度有效 | ✓ | ✗ |
| 被撞会移动 | ✓ | ✗ |
| 能阻挡其他物体 | ✓ | ✓ |
| 适合 | 角色、子弹 | 地板、墙壁 |

## 4. Body 的大小和偏移（Size and Offset）

Physics Body 的碰撞框默认和 Sprite 图片一样大。但有时候图片和你希望的碰撞范围不一致。

**典型问题：** 角色图片周围有透明空白，导致碰撞框比"看起来的角色"大。

```
图片实际内容：
┌──────────────────────┐
│  (透明区域)           │
│    ┌──────────┐      │
│    │  角色图  │      │
│    └──────────┘      │
│  (透明区域)           │
└──────────────────────┘
     ↑ 默认 Body 大小 = 整张图片（包括透明部分）

玩家感知：
角色没碰到敌人，却已经判定碰撞了（因为透明边距也算 Body）
```

解决方法：**手动设置 Body 的大小和偏移**：

```javascript
// 设置 Body 大小（宽, 高）
player.body.setSize(32, 48);

// 设置 Body 偏移（让 Body 在图片内部居中）
player.body.setOffset(16, 8);
```

`setOffset(x, y)` 是 Body 相对于 Sprite 左上角的偏移量，用来对齐 Body 和图片里的实际内容。

---

## 5. 给 Sprite 启用物理

总结几种常见的创建方式：

```javascript
// 方式一：创建带 Dynamic Body 的 Sprite（最常用）
const player = this.physics.add.sprite(x, y, 'key');

// 方式二：创建带 Static Body 的 Sprite
const ground = this.physics.add.staticSprite(x, y, 'key');

// 方式三：已有 Sprite，后来才启用物理
const sprite = this.add.sprite(x, y, 'key');
this.physics.add.existing(sprite);           // Dynamic
this.physics.add.existing(sprite, true);     // Static（第二个参数 true = static）
```

也可以给整组对象统一添加物理（用 Group），但那是进阶用法，本节不要求。

## Self-Check

**Q1.** Physics Body 存储了哪些数据？它和 Sprite 是什么关系？

> **答：** Physics Body 存储物体的速度、加速度、重力影响、碰撞边界大小，以及是否被阻挡（blocked）等物理状态。它和 Sprite 是绑定关系：物理引擎每帧更新 Body 的位置数据，然后 Phaser 自动把 Body 的位置同步到 Sprite 的显示坐标。你移动的是 Body，Sprite 图片跟着走。

---

**Q2.** 下面场景里，地板应该用 Static Body 还是 Dynamic Body？玩家角色呢？说明理由。

> **答：**
> - **地板 → Static Body**：地板永远不动，不受重力影响，只需要阻挡角色下落。Static Body 不需要物理引擎每帧更新，性能开销极低。
> - **玩家角色 → Dynamic Body**：角色需要受重力影响（会下落）、响应速度指令（移动、跳跃）、被地板阻挡。这些都是 Dynamic Body 的特性。

---

**Q3.** 为什么角色的 Physics Body 大小有时候需要手动调小，不能直接用图片原始大小？

> **答：** 角色图片通常有透明边距（空白区域），默认 Body 大小 = 整张图片尺寸，包含这些透明部分。这会导致碰撞判定比"看起来的角色"大，玩家会感觉"明明没碰到，却已经碰撞了"，体验很差。手动用 `setSize()` 缩小 Body，让碰撞范围只覆盖角色图片里的实际内容，配合 `setOffset()` 对齐，才能让碰撞感觉准确。
